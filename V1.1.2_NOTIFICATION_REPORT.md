# WILMS v1.1.2 Notification Report

**Branch:** `hotfix/v1.1.2-user-management-notifications`  
**Version:** `1.1.2`

## Executive summary

Production notification delivery was fragmented: test email worked via Vercel while invitation and domain-event emails still attempted SMTP from Railway (blocked). SMS helpers existed but lacked persistent delivery logging. Registration officers could not view their own borrower details.

This hotfix centralizes outbound mail dispatch, adds delivery logging, wires core domain notifications, and fixes registration UX/RBAC gaps.

## Root causes

| Issue | Root cause |
|-------|------------|
| Invitation emails not received | `createUser` sent mail via Railway SMTP (blocked); errors swallowed with `void` + `console.error` |
| Test email 502 (v1.1.1 follow-up) | Gmail SMTP from Railway times out; fixed in v1.1.1 for test only |
| No delivery audit trail | No `message_deliveries` table; mock-only delivery log in frontend |
| Registration officer cannot view | View link pointed to super-admin `/borrowers/:id` shell |
| Address field instability | `watch()` without scope re-rendered entire wizard on every keystroke |
| Type of Work "Other" | No conditional `typeOfWorkOther` field or validation |

## Changes

### Mail & delivery infrastructure

- `apps/backend/src/infrastructure/mail/dispatch.ts` — Vercel relay for Gmail when `WILMS_VERCEL_MAIL_URL` + `WILMS_INTERNAL_MAIL_SECRET` set
- `apps/frontend/src/app/api/mail/send/route.ts` — server-to-server Gmail relay endpoint
- `apps/backend/src/infrastructure/notifications/delivery-log.ts` — persistent delivery logging
- `apps/backend/src/db/migrations/0014_v112_notifications.sql` — `message_deliveries` table
- `GET /settings/delivery-logs` — searchable delivery history for admins

### Registration fixes

- Officer detail page: `/officer/my-registrations/[id]`
- RBAC: officers can only read borrowers they registered
- Workflow status aligned with frontend (`SUBMITTED`, `UNDER_REVIEW`, etc.)
- Draft registrations merged into My Registrations list
- Type of Work "Other" → required "Please specify" field
- Review step no longer subscribes to all form fields

## Production configuration required

**Railway (API):**
- `WILMS_VERCEL_MAIL_URL=https://wilms.vercel.app`
- `WILMS_INTERNAL_MAIL_SECRET=<shared-secret>`
- `WILMS_APP_URL=https://wilms.vercel.app`

**Vercel (frontend):**
- `GMAIL_USER`, `GMAIL_APP_PASSWORD`, `MAIL_FROM`
- `WILMS_INTERNAL_MAIL_SECRET=<same-shared-secret>`

## Remaining limitations

- Password reset / invite token flow not implemented (static temp password `ChangeMe1!`)
- Account activated/disabled, payment reversal, loan rejected/closed emails not yet wired
- Payment reminder scheduler not implemented (`paymentReminderDaysBefore` stored only)
- In-app inbox not populated for all domain events (staff notifications via `sendNotification` API only)
