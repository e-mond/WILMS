## Investigation

### Environment

- Stopped processes on ports 3000 and 3001 before CI runs.
- `playwright.config.ts`: `reuseExistingServer: !process.env.CI` — CI forces fresh `npm run dev` on port 3001.
- Login route `/login` is public; middleware allows unauthenticated access (`src/lib/auth/middleware.ts`).

### Root cause 1 — Bootstrap ordering (fixed)

**Problem:** `LoginForm` is rendered only after `AppBootstrap` completes (`authHydrated && themeHydrated && appLockHydrated`). While splash is shown, the `Sign in` heading is not in the DOM.

**Evidence:** `src/components/providers/AppBootstrap.tsx` returns `WilmsSplashScreen` until all three stores hydrate. `ThemeProvider` sets `html[data-theme-store-ready="true"]` when theme store hydrates.

**Old helper:** Waited for `Sign in` heading first (5s default timeout) → failed during splash.

**Fix:** `e2e/helpers/auth.ts` — wait for `html[data-theme-store-ready="true"]` (30s), then `Sign in` heading and `form[data-login-ready="true"]`.

**Verification:** `splash-bootstrap.spec.ts`, `login-ux.spec.ts`, login axe scan — **pass** after fix.

### Root cause 2 — Mandatory app lock gate (fixed)

**Problem:** After successful login, `AppLockRequiredGate` blocks all authenticated routes until a six-digit PIN is configured (`src/components/auth/AppLockRequiredGate.tsx`). Post-login tests (shell navbar, role journeys, authenticated axe scans) timed out waiting for shell landmarks.

**Evidence:** Login-only tests passed (URL check only). Shell tests failed at `navigation { name: 'Super Admin' }`, `Borrower Status` heading, etc. Subset run before fix: sign-in succeeded but shell never rendered.

**Fix:** `completeMandatoryAppLockIfPresent()` in `signIn()` — detects overlay, clicks **Set up app lock**, enters PIN `123456` twice via digit pad.

**Verification (CI=1, desktop subset):**

| Test | Result |
| ---- | ------ |
| auth-and-theme › super admin navigation | pass |
| role-journeys (3 tests) | pass |
| shell-navbar › sidebar collapse | pass |

### Other config change

- `playwright.config.ts`: `expect.timeout` raised to 15_000ms for cold-start tolerance.
