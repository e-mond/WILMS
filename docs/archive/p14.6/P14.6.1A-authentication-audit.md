## 1. Authentication architecture

### 1.1 Endpoints

| Layer | Path | Method | Purpose |
|-------|------|--------|---------|
| BFF (Vercel) | `/api/auth/csrf` | GET | Issue `wilms_csrf` cookie |
| BFF (Vercel) | `/api/auth/login` | POST | Validate CSRF → call upstream → set `wilms_session` |
| BFF (Vercel) | `/api/auth/logout` | POST | Clear session |
| BFF proxy | `/api/wilms/*` | * | Proxy to Railway; mutations require CSRF |
| API (Railway) | `/auth/login` | POST | Credential check, returns JWT `token` |
| API (Railway) | `/auth/session` | GET | Session probe |

**Production login URL (users):** `https://wilms.vercel.app/login`  
**BFF login API:** `POST https://wilms.vercel.app/api/auth/login`  
**Upstream (from BFF):** `POST https://wilms-production.up.railway.app/auth/login`

### 1.2 Session & CSRF

| Control | Implementation | Evidence |
|---------|----------------|----------|
| Session cookie | `wilms_session` (HttpOnly) | `apps/frontend/src/app/api/auth/login/route.ts` |
| CSRF cookie | `wilms_csrf` | `apps/frontend/src/lib/auth/csrf-server.ts` |
| CSRF header | `x-wilms-csrf` must match cookie | `rejectInvalidCsrf()` → **403** if missing/mismatch |
| API session | HMAC-signed Bearer token | `/health` → `session.provider: hmac-signed-token` |
| CORS | `WILMS_CORS_ORIGIN=https://wilms.vercel.app` | `P14.6-environment-and-credentials.md` |

### 1.3 Login flow (BFF)

```
1. GET /api/auth/csrf        → Set-Cookie: wilms_csrf=...
2. POST /api/auth/login      → Header x-wilms-csrf + Cookie wilms_csrf
   → authenticateCredentials() → apiClient POST /auth/login (Railway)
   → On success: Set-Cookie wilms_session + JSON { user, expiresAt }
3. Subsequent /api/wilms/*   → BFF adds Authorization: Bearer <session>
```

### 3.3 Password validation (code path)

- `verifyPassword` fails → **401** (same message)
- Success → **200** with `token`, `userId`, `role`

`verifyPassword` supports bcrypt hash or plaintext (dev) — `apps/backend/src/lib/password.ts`.

## 4. Cross-layer auth risk (deployment drift)

| Scenario | Risk |
|----------|------|
| Vercel on v0.2.2, Railway on v0.2.1 | Frontend may expect sync/offline routes not on API; core login **should** still work if users unchanged |
| Railway stuck on v0.2.1 | P14.6 login 401 fix on frontend deployed; backend unchanged — **unlikely** to cause 401 by itself |
| Demo users suspended/rotated | **401** on valid-looking credentials |
| Missing CSRF on `POST /api/auth/login` | **403** (not 401) |
