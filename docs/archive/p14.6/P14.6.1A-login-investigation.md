# P14.6.1A — Login Failure Investigation (HTTP 401)

**Date:** 2026-06-30  
**Mode:** Read-only  
**Reported symptom:** `POST /api/auth/login` → HTTP **401**  
**Live reproduction:** **Not completed** (curl/Node hung in audit environment)

---

## 1. Endpoint map

| Client calls | Handler | Typical status on failure |
|--------------|---------|---------------------------|
| `POST /api/auth/login` | Next.js BFF `login/route.ts` | **401** invalid creds · **403** CSRF · **422** validation |
| `POST /api/wilms/auth/login` | BFF proxy → Railway | **403** CSRF (proxy) · upstream **401** |
| `POST https://wilms-production.up.railway.app/auth/login` | Express `auth/routes.ts` | **401** invalid creds · **422** validation |

**Note:** Production UI uses **`/api/auth/login`** (BFF), not `/api/wilms/auth/login`.

---

## 2. Response semantics (code)

### 2.1 BFF — `apps/frontend/src/app/api/auth/login/route.ts`

| Condition | Status | Body |
|-----------|--------|------|
| CSRF invalid/missing | **403** | `{ "message": "Invalid CSRF token." }` |
| JSON parse error | **422** | `{ "message": "Invalid request body." }` |
| Zod validation fail | **422** | first issue message |
| `authenticateCredentials()` returns null | **401** | `{ "message": "Invalid email or password." }` |
| Success | **200** | `{ user, expiresAt }` + `Set-Cookie: wilms_session` |

### 2.2 BFF authenticate — `apps/frontend/src/lib/auth/authenticate.ts`

Returns `null` when:

- `apiClient.post('/auth/login')` throws (network, 401, 5xx), or
- Response has no `token` field

### 2.3 API — `apps/backend/src/modules/auth/routes.ts`

| Condition | Status | Body |
|-----------|--------|------|
| User missing | **401** | `Invalid email or password.` |
| `user.status === 'SUSPENDED'` | **401** | same (no enumeration) |
| Password mismatch | **401** | same |
| Success | **200** | `{ data: { token, userId, role, ... } }` |

Direct probe:

```
GET https://wilms-production.up.railway.app/auth/login
→ 401 Unauthorized (method not allowed / unauthenticated — confirms route exists)
```

## 3. Request payload (expected)

```json
{
  "email": "admin@wilms.demo",
  "password": "DemoAdmin1!"
}
```

Headers for BFF success path:

```
Content-Type: application/json
Cookie: wilms_csrf=<token>
x-wilms-csrf: <same token>
```

### Minimal reproduction (operator)

```bash
# 1. CSRF
curl -c cookies.txt -b cookies.txt https://wilms.vercel.app/api/auth/csrf

# 2. Login (extract wilms_csrf from cookies.txt)
curl -b cookies.txt -c cookies.txt \
  -H "Content-Type: application/json" \

  -d '{"email":"admin@wilms.demo","password":"DemoAdmin1!"}' \
  https://wilms.vercel.app/api/auth/login -w "\nHTTP:%{http_code}\n"
```

## 4. Classification matrix

| Observation | Classification | Likelihood |
|-------------|----------------|------------|
| **401** on `/api/auth/login` **without** CSRF cookie/header | **Expected** if caller mistakes 403 for 401 | Low — should be **403** |
| **401** with valid CSRF + seed credentials | **Environment** or **Database** | Medium — user missing/suspended/wrong hash |
| **401** after demo rotation script | **Environment** | Medium — passwords changed to `*@wilms.production` |
| **401** only from browser, curl works | **Configuration** (cookie domain / Secure) | Low |
| **401** all roles, API v0.2.1 | **Database** seed drift | Medium |
| Vercel v0.2.2 + Railway v0.2.1 causes 401 | **Deployment drift** | Low for login path |
| Upstream timeout → BFF 401 | **Deployment** / **Application** | Low |

### Primary classification (pending live body capture)

**Most likely: Environment / Database** — credentials in use do not match production `users` table (rotation, suspension, or typo).

**Second: Expected behavior** — conflating **403 CSRF** with **401** if CSRF step skipped in manual API testing.

**Not primary: Application bug in P14.6 login fix** — Railway is still on **v0.2.1**; P14.6 BFF changes may be on Vercel without matching API, but login path is unchanged on v0.2.1 backend.

## 6. Session creation path

On **200**:

1. Railway returns `token` in `data`
2. BFF `authenticateCredentials` maps to `sessionToken`
3. BFF sets `wilms_session` HttpOnly cookie
4. CSRF cookie refreshed via `setCsrfCookie`

On **401**: no session cookie; client shows generic error (`authService.ts`).

### Provisional root cause statement

> HTTP **401** on `POST /api/auth/login` indicates the BFF rejected credentials after upstream auth failure or missing token — **not** a missing merge of P14.6. If CSRF was omitted, expect **403** instead. With deployment drift (Railway v0.2.1, Vercel possibly v0.2.2), verify credentials against production DB before blaming application code.
