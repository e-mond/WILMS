## Policy

---

## P14.6 Sync Specifics

| Path | Safety mechanism |
|------|------------------|
| Financial ingest | Single TX: operation row + conflict row atomically |
| Approve | TX wraps conflict resolution + `recordPayment` + status updates |
| Reject | TX wraps conflict + operation status — no payment call |
| Idempotency | Unique index prevents duplicate operations outside TX race window |
| Non-financial ingest | Single insert (no financial side effects today) |

## Migration Safety

`0007_offline_sync.sql` — additive only (two new tables, indexes). No ALTER on existing financial tables. Rollback: `DROP TABLE offline_sync_conflicts; DROP TABLE offline_sync_operations;`
