## Race Condition ÔÇö Concurrent Same Key

**Scenario:** Two parallel requests insert same idempotency key.

**Before P14.3A.1:** Second insert could throw unhandled PostgreSQL 23505 ÔåÆ HTTP 500.

**After P14.3A.1:** Unique violation caught ÔåÆ re-read row ÔåÆ `IDEMPOTENCY_IN_PROGRESS` or replay.

**Evidence:** `isUniqueViolation()` + `resolveExistingIdempotency()` in `idempotency.repository.ts`.
