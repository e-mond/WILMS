## Previous failure diagnosis

**Error:** `The Next.js output directory ".next" was not found`

### Root cause (confirmed with evidence)

| # | Investigation | Finding | Evidence |
|---|---------------|---------|----------|
| 1 | Where does `npm run build` write output? | `apps/frontend/.next/` | `Test-Path apps/frontend/.next` ÔåÆ `True` after build; `package.json` L12 runs `build -w @wilms/frontend` |
| 2 | Where does Vercel look for `.next`? | Relative to **Root Directory** | Vercel Next.js preset expects `.next` in project root |
| 3 | Monorepo root build | No `.next` at repo root | Build output only under `apps/frontend/` |
| 4 | `turbo.json` outputs | Declares `.next/**` for turbo cache | `turbo.json` L6 ÔÇö correct for Turbo, does not relocate Vercel output |
| 5 | `next.config.mjs` | No `distDir` override; no `output: 'standalone'` | `apps/frontend/next.config.mjs` L61ÔÇô109 |
| 6 | `vercel.json` before fix | **Absent** | No prior monorepo routing |

**Conclusion:** Vercel Root Directory was the monorepo root (or misconfigured), while Next.js emits `.next` inside `apps/frontend/`. Path mismatch causes the error.
