## Problem

The reconciliation UI was not demo-ready:

1. **Variance mismatch** ÔÇö preview used `physical ÔêÆ actual_collected` while the backend stores `physical ÔêÆ expected_due`, causing stakeholder confusion when submitted values differed from the preview.
2. **Missing comment field** ÔÇö flagged variance submissions failed with HTTP 422 because the backend requires a ÔëÑ10 character explanation.
3. **Misleading errors** ÔÇö HTTP 403 (permission denied) displayed the same "session expired" message as 401.
4. **Dev mock lock-in** ÔÇö `npm run dev` could not reach the certified backend without a production build.

---

## Root Cause

- Frontend duplicated variance logic with the wrong operand (`actualPesewas` instead of `expectedPesewas`).
- Submit payload omitted `comment`; no client-side validation for flagged cases.
- `apiClient.ts` mapped both 401 and 403 to session-expired messaging.
- `resolveDataProviderMode()` and webpack aliases forced mock mode whenever `NODE_ENV !== 'production'`.

---

## Changes Made

### Phase 1 ÔÇö Variance alignment

- `calculatePrimaryVariancePesewas(physical, expectedDue)` in `reconciliation-summary.ts`.
- `ReconciliationForm` preview and mock service submit use expected due.
- Unit tests for zero / positive / negative variance.

### Phase 2 ÔÇö Flagged comment support

- Textarea shown when preview variance exceeds threshold.
- `flaggedCommentSchema` + `RECONCILIATION_FLAGGED_COMMENT_MIN_LENGTH = 10`.
- Comment included in submit mutation; mock service enforces same rule.

### Phase 3 ÔÇö 401 vs 403

- `API_ERROR_CODE.FORBIDDEN` for 403.
- Distinct user-facing messages; session redirect only on 401.

### Phase 4 ÔÇö Dev live API

- `NEXT_PUBLIC_USE_MOCK=false` + API URL switches to live data provider in development.
- Documented in `.env.local.example`.

### Phase 5 ÔÇö Proxy validation tooling

- `apps/frontend/scripts/proxy-reconciliation-smoke.mjs` exercises login ÔåÆ cookie ÔåÆ proxy ÔåÆ GET/POST ÔåÆ duplicate ÔåÆ RBAC denial.
