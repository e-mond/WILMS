## Can developers switch live/mock without rebuild?

| Scenario | Works? | Evidence |
|----------|--------|----------|
| `npm run dev` + default env | Mock | `shouldUseMockServices()` ÔåÆ true (NODE_ENV development, USE_MOCK unset) |
| `npm run dev` + `USE_MOCK=false` + API URL | **Live** | `shouldUseMockServices()` ÔåÆ false at dev server start |
| Change env without restarting `next dev` | **No** | Webpack alias resolved at compile time |
| `npm run build` + `start` with production env | Live (if URL set) | `NODE_ENV=production` |

**Rating:** PARTIALLY meets desired behavior ÔÇö switch requires **restart** of dev server, not rebuild. No hot env toggle.

## Live mode service coverage

When `ApiDataProvider` active, all services export from `*.ts` files using `apiClient`. Backend routes exist only for:

`auth`, `borrowers`, `loans`, `payments`, `adjustments`, `loan-pools`, `group-formation`, `reconciliation`, `reports`, `uploads`, `audit`, `health`

**No backend** for API calls from: `dashboardService`, `collectorService`, `groupService`, `notificationService`, `searchService`, `riskFlagService`, `settingsService`, `expenseService`, `collectionMetricsService`, `overpaymentReviewService`, `locationService`, `photoCaptureSessionService` (partial ÔÇö no capture-sessions routes found), `collectorManagementService`.

Live mode for those domains ÔåÆ **HTTP 404** unless BFF/mock fallback added.

## Consolidation recommendation (design only ÔÇö do not implement)

### Proposed: `NEXT_PUBLIC_APP_MODE=live | mock | demo`

| Criterion | Assessment |
|-----------|------------|
| Covers current combinations | **Partial** ÔÇö `demo` vs `mock` undefined; export label separate |
| Backward compatibility | Breaking unless shim reads old flags |
| Migration effort | **M** (0.5ÔÇô2 days) |
| Regression risk | **Medium** ÔÇö affects all service imports |
| Meets single-switch goal | **Yes** if `live` maps to current `USE_MOCK=false` + URL |
| **Recommendation** | **Defer** to post-v1; document current flag precedence in README (done). Optional v1.1: add `APP_MODE` reading legacy flags as fallback |

### Flags to retain short-term

| Flag | Retain? | Reason |
|------|---------|--------|
| `NEXT_PUBLIC_USE_MOCK` | **Yes** | Primary live switch |
| `NEXT_PUBLIC_API_BASE_URL` | **Yes** | Required |
| `NEXT_PUBLIC_DEMO_MODE` | **Yes** | CI/E2E may use |
| `NEXT_PUBLIC_FORCE_DEMO_MODE` | Consolidate later | Redundant |
| `NEXT_PUBLIC_API_DISABLED` | Consolidate later | Redundant |
