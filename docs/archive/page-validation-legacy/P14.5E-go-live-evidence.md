## 2. Railway Deployment Report

### Root cause of initial failures (fixed)

1. **Deploy from `apps/backend`** ÔÇö only 161 KB uploaded; `cd ../.. && npm ci` had no monorepo context.
2. **Duplicate `npm ci` in build** ÔÇö Nixpacks cache EBUSY.
3. **DevDependencies omitted** ÔÇö `NODE_ENV=production` skipped `tsx` / `drizzle-kit`.
4. **Auto `turbo build`** ÔÇö Nixpacks detected monorepo and ran `npx turbo run build` (failed).

### Resolution

- **`Dockerfile`** at repo root ÔÇö `npm ci --include=dev`, `start:prod -w @wilms/api`
- **`railway.toml`** ÔÇö `builder = "DOCKERFILE"`, healthcheck `/health`
- Deploy from **monorepo root** (`railway up`)
- Production env vars set on Railway service

## 3. Vercel Deployment Report

### Root cause of initial failures (fixed)

1. **CLI deploy from `apps/frontend`** ÔÇö partial upload, no root `package-lock.json`.
2. **Project Root Directory `.` with `cd ../.. && npm ci`** ÔÇö install ran outside repo.
3. **Missing persisted env vars** ÔÇö mock mode active (`NEXT_PUBLIC_API_BASE_URL` unset).

### Resolution

- **`vercel.json`** at repo root:
  - `installCommand`: `npm ci`
  - `buildCommand`: `npm run build -w @wilms/frontend`
  - `outputDirectory`: `apps/frontend/.next`
- Deploy from **monorepo root**
- Production env vars added via `vercel env add`:
  - `NEXT_PUBLIC_API_BASE_URL=https://frontend-ashen-gamma-11.vercel.app/api/wilms`
  - `WILMS_API_UPSTREAM=https://wilms-production.up.railway.app`
  - `NEXT_PUBLIC_USE_MOCK=false`
  - `NEXT_PUBLIC_APP_URL`, `NEXT_PUBLIC_WILMS_ENV=production`
