## Demo Sequence (13 Steps)

### Step 1 ÔÇö Opening & system context
**Time:** 2 min | **Role:** Presenter

- State purpose: live Women's Interest-Free Loan Management System on certified backend.
- Note: API evidence primary; browser walkthrough is rehearsal ÔÇö capture screenshots here if needed.
- Confirm both servers running and live API mode active.

---

### Step 2 ÔÇö Super Admin login
**Time:** 1 min | **Role:** Super Admin (`admin@wilms.demo`)

- Navigate to `http://127.0.0.1:3000/login`.
- Sign in as Super Admin.
- **Show:** Dashboard loads; session cookie set (no mock data banner if live mode correct).
- **API evidence:** `authentication/login-admin` PASS (~1253 ms first login).

---

### Step 3 ÔÇö Borrower search
**Time:** 2 min | **Role:** Super Admin

- Open borrowers list; search for a name (e.g. "Ama" or "Akosua").
- **Show:** Search returns approved borrowers with group names.
- **Talking point:** Registration approval workflow; approved borrowers eligible for loans.
- **API evidence:** `borrower/search` PASS, 197 ms.

---

### Step 4 ÔÇö Borrower profile
**Time:** 1 min | **Role:** Super Admin

- Open borrower detail (e.g. Akosua Mensah, Sunrise Group).
- **Show:** ID type, phone, community, registration date, status APPROVED.
- **API evidence:** `borrower/profile` PASS, 201 ms.

---

### Step 5 ÔÇö Loan lifecycle (create ÔåÆ approve ÔåÆ disburse)
**Time:** 3 min | **Role:** Loan Approver (`approver@wilms.demo`)

- **If inline create works:** Create loan for eligible borrower ÔåÆ approve ÔåÆ disburse.
- **If seed exhausted (expected on shared Neon):** Explain validation ÔÇö "borrower already has active/pending loan." Show existing active loan schedule instead.
- **Fallback narrative:** Same-session `verify:live` certified create ÔåÆ approve ÔåÆ disburse ÔåÆ payment (23/23).
- **API evidence:** Inline `loan/create` **FAIL**; carried `verify:live workflow-create-loan` **PASS**.

---

### Step 6 ÔÇö Collection recording
**Time:** 2 min | **Role:** Collector (`collector@wilms.demo`)

- Switch to Collector account (logout ÔåÆ login, or second browser profile).
- Record a field payment against an active loan (collection sheet or payment form).
- **Show:** Full weekly amount; GPS capture if on collection sheet route.
- **Fallback:** Reference `verify:live` `workflow-post-repayment` if no fresh loan available.
- **Talking point:** No partial payments; oldest obligation cleared first (BRD ┬º9.1).

---

### Step 7 ÔÇö Reconciliation summary
**Time:** 2 min | **Role:** Collector

- Navigate to `/collector/reconciliation`.
- Select a date with zero expected collections (or use cert date pattern).
- **Show:** Expected due, system recorded, physical cash preview ÔÇö variance formula matches backend (`physical ÔêÆ expected`).
- **API evidence:** `reconciliation/fetch-summary` PASS, 939 ms.

---

### Step 8 ÔÇö Submit zero-variance reconciliation
**Time:** 2 min | **Role:** Collector

- Enter physical cash matching expected due; submit.
- **Show:** Success confirmation; submitted state.
- **Talking point:** Flagged variance requires ÔëÑ10 character comment (4D.1 UI fix).
- **API evidence:** `reconciliation/submit-zero-variance` PASS, 1550 ms.

---

### Step 9 ÔÇö Duplicate submission blocked
**Time:** 1 min | **Role:** Collector

- Attempt second submit for same date.
- **Show:** Validation error ÔÇö "Reconciliation already submitted for this date."
- **API evidence:** `reconciliation/duplicate-prevention` PASS, HTTP 422.

---

### Step 10 ÔÇö Document upload (local provider)
**Time:** 2 min | **Role:** Super Admin or Officer

- Upload borrower photo or registration attachment.
- **Show:** Upload succeeds; preview loads from `/uploads/{id}/content`.
- **Disclose:** Cloudinary **not configured** ÔÇö local disk + PostgreSQL for demo; production will use CDN when creds set.
- **API evidence:** All upload steps PASS; `uuidv7()` ID fix in `upload.service.ts`.

---

### Step 11 ÔÇö Payment reversal
**Time:** 3 min | **Role:** Super Admin

- Open payment requiring reversal; execute reversal with reason.
- **Show:** Balance restored; duplicate reversal rejected.
- **Fallback narrative:** `cert:reversal:functional` 10/10 if inline prep fails.
- **API evidence:** `reversal/successful-reversal` PASS; execute ~3994 ms.

---

### Step 12 ÔÇö RBAC enforcement
**Time:** 1 min | **Role:** Officer (`officer@wilms.demo`)

- Sign in as Officer; attempt reconciliation submit or restricted admin action.
- **Show:** HTTP 403 ÔÇö "You do not have permission" (not "session expired" ÔÇö 4D.1 fix).
- **API evidence:** `authentication/rbac-officer-denied` PASS, 3 ms.

---

### Step 13 ÔÇö Logout & close
**Time:** 2 min | **Role:** Presenter

- Logout via UI (clears `wilms_session` cookie via `/api/auth/logout`).
- Summarize certified workflows: borrowers, reconciliation, uploads (local), reversals, RBAC.
- Open Q&A; reference `P14.3B-phase-4d2-functional-certification.md` for evidence detail.
