### Provider selection

| `UPLOAD_PROVIDER` | Credentials | Active provider |
|-------------------|-------------|-----------------|
| `local` (default) | ÔÇö | `LocalUploadProvider` |
| `cloudinary` | All `CLOUDINARY_*` set | `CloudinaryUploadProvider` |
| `cloudinary` | Missing creds | Falls back to local |

### Signed uploads

`GET /uploads/signature` (auth required) returns Cloudinary signed params when configured:

```json
{
  "data": {
    "cloudName": "...",
    "apiKey": "...",
    "timestamp": 1234567890,
    "signature": "...",
    "folder": "wilms"
  }
}
```

Uses `cloudinary.utils.api_sign_request`.

### Validation

- `UPLOAD_MAX_SIZE_BYTES` (default 10 MB)
- `UPLOAD_ALLOWED_MIME_TYPES` (comma-separated)

Applied in `POST /uploads` before save.
