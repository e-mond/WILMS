## DATABASE_URL

### Requirement

Backend reads `process.env.DATABASE_URL` directly ÔÇö **no dotenv loader** in `apps/backend/src/` (`config/env.ts`, `db/client.ts`).

When unset, backend uses **in-memory** persistence (`db/persistence.ts`, startup log: `in-memory`).

When set, backend uses **PostgreSQL via Neon** (`@neondatabase/serverless` + Drizzle).

### Present in `.env.example`

```env
DATABASE_URL=postgresql://ÔÇª@ÔÇªneon.tech/neondb?sslmode=require&channel_binding=require
```

`DATABASE_URL` **exists** in root `.env.example` with a Neon pooler connection string.

**Security note:** `.env.example` contains what appears to be a live Neon password. Treat as exposed; rotate in Neon console and replace example with a placeholder before public distribution.

### Missing from backend-specific config

There is no `apps/backend/.env.example`. Developers must supply `DATABASE_URL` via:

1. Copy root `.env.example` ÔåÆ `.env` at repo root, **or**
2. Export `DATABASE_URL` in the shell before backend/drizzle commands, **or**
3. Set `DATABASE_URL` in the deployment platform (Neon integration, Vercel, etc.)

Backend does **not** auto-load `.env` ÔÇö the variable must be in the process environment when running `npm run dev:api`, `db:migrate`, `db:seed`, or `verify:financial`.

**Mock vs API resolution** (`resolveDataProviderMode()`):

## What the Developer Must Do Next

1. **Create `.env` at repo root** (or use a tool that loads it into the shell):

   ```powershell
   Copy-Item .env.example .env
   ```

2. **Set a valid Neon `DATABASE_URL`** in `.env` (from Neon dashboard ÔåÆ Connection string ÔåÆ pooled). Rotate any credential that was committed in `.env.example`.

3. **Load env before backend commands** (PowerShell example):

   ```powershell
   Get-Content .env | ForEach-Object {
     if ($_ -match '^\s*([^#][^=]+)=(.*)$') {
       [System.Environment]::SetEnvironmentVariable($matches[1].Trim(), $matches[2].Trim(), 'Process')
     }
   }
   ```

4. **Apply schema and seed:**

   ```bash
   npm run db:migrate -w @wilms/api
   npm run db:seed -w @wilms/api
   ```

   > **Known defect (2026-06-09):** `db:seed` fails on RBAC insert ÔÇö see `P14.3A.2-neon-verification.md`.

5. **Verify persisted financial layer:**

   ```bash
   npm run verify:financial -w @wilms/api
   ```

6. **For live frontend Ôåö backend integration** (not mock):

   - Run backend with `DATABASE_URL` set: `npm run dev:api`
   - Set `NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:3001/api/wilms` (BFF) or direct backend URL
   - Set `WILMS_API_UPSTREAM=http://127.0.0.1:4000`
   - Build/run frontend with `NODE_ENV=production` **or** change dev mock policy (current code forces mock in development)

7. **Neon credentials:** Required from the developer if `.env.example` credentials are invalid or rotated. Obtain from [Neon Console](https://console.neon.tech) ÔåÆ project ÔåÆ Connect.
