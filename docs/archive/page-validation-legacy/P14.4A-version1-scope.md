### Decisions

1. **RC-001 (scope boundary):** V1 ships the **financial core** and **certified role workflows** (collector payment/reconciliation, officer registration, approver loan/borrower review, super-admin financial pages that are live-API-safe). Evidence: P14.3C ÔÇö 19/46 pages live-API-safe; 8/11 workflows Complete.

2. **Mock-only admin UI (RC-034ÔÇôRC-041):** Deferred to **V2 for live production** because absence does not block certified collector payment or officer registration workflows (P14.3C Feature Coverage ┬ºMock-only). UI remains for mock/demo builds.

3. **Report stubs (RC-019ÔÇôRC-023):** **Ship in V1** with **Conditional** status ÔÇö endpoints exist but return empty data; daily collection and hub are live.

4. **Phase 5C:** Spec references 5C remediation; **no 5C documents or commits exist**. No blocker may be marked Resolved based on 5C.

5. **Spec role names** (Executive, Regional Manager, Branch Manager, System Admin): **Not present** in `packages/shared-rbac/src/roles.ts`. V1 verifies **6 implemented roles** only (see P14.4F).
