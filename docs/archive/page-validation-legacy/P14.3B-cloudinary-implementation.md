```text
                                          Ôö£ÔöÇÔöÇ validateUploadInput()

                                          ÔööÔöÇÔöÇ upload.repository.insertUpload()  ÔåÆ PostgreSQL (when DATABASE_URL set)

```

## Components

### Upload service (`upload.service.ts`)

- `saveUpload()` ÔÇö generates ID, stores blob, persists metadata
- `getUploadRecord()` ÔÇö DB first, memory fallback
- `resolveUploadAccessUrl()` ÔÇö CDN URL vs API proxy path
- `readUploadBuffer()` ÔÇö for local proxy route

### Environment validation (`env-validation.ts`)

- Production + `UPLOAD_PROVIDER=cloudinary` without creds ÔåÆ **error**
- Development + missing creds ÔåÆ **warning**, fallback to local
- Production + `UPLOAD_PROVIDER=local` ÔåÆ **warning**

### Transformation helper (`cloudinary-transform.ts`)

| Preset | Use case | Transform |
|--------|----------|-----------|
| `profile-thumb` | Staff profile | 200├ù200 face crop |
| `borrower-photo` | Borrower/guarantor | 400├ù400 face crop |
| `document-preview` | Documents | 1200px width limit |
| `original` | Signatures, thumbprints | No transform |

### Delete utility (`delete-upload.ts`)

1. Load metadata
2. Delete from Cloudinary / local disk
3. Soft-delete row (`deleted_at`) when DB enabled

## Security Controls

- `CLOUDINARY_API_SECRET` read only in backend `config.ts` and `cloudinary-provider.ts`
- `cert-upload-environment.ts` masks credentials in output
- `GET /uploads/signature` returns `apiKey` + `signature` only (standard Cloudinary pattern)
- All upload routes require `requireAuth`
- No secrets in frontend bundles or API error responses
