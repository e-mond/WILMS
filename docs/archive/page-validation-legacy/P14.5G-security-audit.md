## CSRF Protection

| Control | Implementation |
|---------|----------------|
| Token storage | Non-HttpOnly cookie `wilms_csrf` (readable by JS for double-submit) |
| Token validation | BFF compares cookie vs `x-wilms-csrf` header |
| Protected routes | `/api/auth/login`, `/api/auth/logout`, `/api/wilms/*` mutations |
| Token issuance | `GET /api/auth/csrf` |
| Rotation | New token on successful login |
| Secure flag | `secure: true` in production |

**Residual risk:** Express API on Railway is not CSRF-protected directly; browser calls go through BFF with session cookies. Direct API access requires session cookie (SameSite=Lax mitigates cross-site POST).
