## Summary

Repayment posting applies frontend-identical allocation rules inside a single database transaction: payment record, schedule week PAID, loan balance update, ledger entry, audit log.

---

## Allocation Rules (frontend parity)

| Rule | Implementation |
|------|----------------|
| Exact weekly amount only | `validatePaymentSubmission()` rejects partial/overpayment |
| Payment day gate | Weekday of `paymentDate` must match loan `paymentDay` |
| Oldest obligation first | `getOldestPayableWeek()` ÔÇö MISSED or PENDING with `dueDate <= referenceDate` |
| No advance payments | Rejected when no payable week exists |

**Evidence:** `apps/backend/src/domain/payment/allocation.ts` ÔÇö mirrors `payment-entry.utils.ts` / `paymentService.mock.ts`.

---

## Schedule Generation

| Rule | Implementation |
|------|----------------|
| Divisibility | `amountPesewas % durationWeeks === 0` enforced at create |
| Weekly amount | `Math.floor(amountPesewas / durationWeeks)` |
| First due date | First occurrence of `paymentDay` on/after `startDate` |
| No remainder absorption | Equal weekly installments only |

**Evidence:** `apps/backend/src/domain/loan/calculations.ts`, `schedule.ts`.

---

## Posting Flow

```
POST /payments (Idempotency-Key optional)
  ÔåÆ validate submission
  ÔåÆ applyMissedWeekMarking (PENDING ÔåÆ MISSED when dueDate < paymentDate)
  ÔåÆ allocate oldest payable week
  ÔåÆ transaction:
       append payment (loanId, scheduleWeekNumber)
       markWeekPaid (optimistic version)
       updateLoanLifecycle (COMPLETED if balance 0)
       appendLedgerEntry (REPAYMENT)
  ÔåÆ audit payment.recorded
```

**Evidence:** `apps/backend/src/modules/payments/service.ts` ÔÇö `recordPayment`, `postPayment`.

---

## Balance Tracking

Outstanding balance stored as `loan_balance NUMERIC(15,2)`; decremented by payment amount pesewas converted at boundary.

**Evidence:** `loan.repository.ts` `updateLoanLifecycle`; `domain/money.ts` `pesewasToDecimal` / `decimalToPesewas`.

---

## Progress Calculation

Server replicates AMB-006 formula from `loan-progress.utils.ts`.

**Evidence:** `apps/backend/src/domain/loan/mappers.ts` ÔÇö `calculateLoanProgress()`.

---

## Determinism

All calculations use integer pesewas at API boundary and fixed rounding via `pesewasToDecimal` (2 dp). Schedule week selection is order-stable (ascending week number).
