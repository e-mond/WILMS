## Frontend route guards

Mechanism: cookie session ÔåÆ role ÔåÆ `resolveUserPermissionIds` ÔåÆ `ROUTE_PERMISSION_REQUIREMENTS` prefix match.

| Prefix | Required permissions (any) |
|--------|---------------------------|
| `/dashboard` | ACCESS_ADMIN_PORTAL, VIEW_REPORTS |
| `/borrowers` | ACCESS_ADMIN_PORTAL, VIEW_ASSIGNED_BORROWERS, REGISTER_BORROWERS |
| `/loans` | ACCESS_ADMIN_PORTAL, APPROVE_LOANS, VIEW_FINANCIAL_REPORTS |
| `/collector` | ACCESS_COLLECTOR_PORTAL |
| `/approver` | ACCESS_APPROVER_PORTAL |
| `/auditor` | ACCESS_AUDITOR_PORTAL |
| `/officer` | ACCESS_REGISTRATION_PORTAL |
| `/settings` | MANAGE_SYSTEM_SETTINGS, MANAGE_USERS |
| `/reports` | VIEW_REPORTS, VIEW_AUDIT_LOG, EXPORT_REPORTS |

**Gap:** Routes with no matching prefix ÔåÆ **denied** (`canAccessPathWithPermissions` returns false for unknown paths). Mock-only pages that match a prefix (e.g. `/dashboard`) are **reachable** by super-admin even when backend missing.

## RBAC inconsistencies

| ID | Issue | Evidence | Severity |
|----|-------|----------|----------|
| RBAC-001 | Frontend approver can reach `/loans` UI but `GET /loans` requires `VIEW_FINANCIAL_REPORTS` which approver **lacks** | `role-permissions.ts` approver set; `loans/routes.ts` L39; verify:live expects 403 for approver list loans | **By design** ÔÇö UI vs API mismatch |
| RBAC-002 | Payment reversal requires `ACCESS_ADMIN_PORTAL` not dedicated reversal permission | `payments/routes.ts` L210 | Low ÔÇö super-admin only |
| RBAC-003 | Upload routes use `requireAuth` only on POST ÔÇö no purpose-specific permission | `uploads/routes.ts` | Medium ÔÇö any authenticated user can upload |
| RBAC-004 | `requirePermission` returns 403 with code UNAUTHORIZED (not FORBIDDEN) | `require-permission.ts` L17 | Low ÔÇö semantic |
| RBAC-005 | Frontend permission overrides via `getUserPermissionOverrides` ÔÇö client-side only | `permission-matrix.ts` | Medium ÔÇö must not trust for API auth (API uses server session role) |

## Verdict

**RBAC implementation is consistent for financial-core workflows** and certified by harness. **UI exposes routes beyond API permission alignment** for approver loan list and mock-only admin pages ÔÇö acceptable under V1 scope if product blocks those routes in live deploy.
