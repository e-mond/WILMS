## New Tables

### 1. `financial_reversals`

| Attribute | Value |
|-----------|-------|
| **Purpose** | Workflow record for each reversal request/execution |
| **Relationships** | FK optional to `payments`, `financial_adjustments`, `loans`, `borrowers`; FK to `users` (requested_by, executed_by) |
| **Expected volume** | Low ÔÇö admin operations; estimate &lt; 0.1% of payment volume |
| **Retention** | Permanent (financial control record) |

**Proposed columns**

| Column | Type | Notes |
|--------|------|-------|
| id | uuid PK | uuidv7 |
| source_type | enum | PAYMENT, ADJUSTMENT, DISBURSEMENT (future) |
| source_id | uuid | Polymorphic target |
| loan_id | uuid FK | Denormalized for queries |
| borrower_id | uuid FK | |
| amount_pesewas | integer | Amount negated |
| reason | text | min 10 chars |
| status | enum | PENDING, EXECUTED, REJECTED (v1 may skip PENDING) |
| requested_by_user_id | uuid FK | |
| requested_at | timestamptz | |
| executed_by_user_id | uuid nullable | |
| executed_at | timestamptz nullable | |
| before_balance_pesewas | integer nullable | Set on execute |
| after_balance_pesewas | integer nullable | |
| delta_pesewas | integer nullable | |
| version | integer | Optimistic lock |
| created_at / updated_at | timestamptz | |

**Constraints**

- Partial unique index: `(source_type, source_id) WHERE status = 'EXECUTED'`
- Check: `amount_pesewas > 0`
