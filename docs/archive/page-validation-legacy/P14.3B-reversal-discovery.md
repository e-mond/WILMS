The WILMS platform has **no dedicated Financial Reversals domain**. Reversal-like behavior appears only as:

1. **Mock UI flows** ÔÇö same-day payment edit, payment-correction adjustments, mock transaction log.

**Discovery verdict:** Reversal **implementation is not ready**. Architecture design (**P14.3B.3B**) may proceed after human PR approval with label `approved-for-3b-architecture`.

## 2. Ledger Architecture Review

### Entry types and creation paths

| Entry type | Created by | Repository | API trigger |
|------------|------------|------------|-------------|
| `INTEREST_CHARGE` | `loans/service.createLoan` | `ledger.repository.appendLedgerEntry` | `POST /loans` |
| `LOAN_DISBURSEMENT` | `loans/service.disburseLoan` | same | `POST /loans/:id/disburse` |
| `REPAYMENT` | `payments/service.postPayment` | same | `POST /payments` |
| `ADJUSTMENT` | `adjustments/service.approveAdjustment` | same | `POST /adjustments/:id/approve` |

**Balance update paths:**

| Operation | Balance field | Mechanism |
|-----------|---------------|-----------|
| Disburse | `loans.loanBalance` = principal | `updateLoanLifecycle` |
| Repayment | Decrements `loanBalance` | `updateLoanLifecycle` in payment txn |
| Adjustment approve | Computed delta | `computeBalanceEffect` + `updateLoanLifecycle` |

**Balance derivation:** Stored on `loans.loanBalance`; verified by `verify:financial` ledger/payment reconciliation checks.

**Reversal implication:** Compensating entries must integrate with loan `version`, schedule week `PAID` status, and payment row state ÔÇö none of which have reverse writers today.

## 3. Adjustment Integration Review

### Can reversals reuse adjustment infrastructure?

| Component | Reuse | Reason |
|-----------|-------|--------|
| `runInTransaction` + optimistic lock pattern | **Yes** | Proven on approve |
| `adjustment_history` before/after/delta | **Partial** | Pattern reusable; semantics differ (approval vs direct reversal) |
| `financial_adjustments` table | **No** | Approval workflow; reversals need source-transaction linkage |
| `PAYMENT_CORRECTION` type | **Needs Alignment** | Overlaps reversal intent but admin-gated and does not undo payment/schedule rows |
