## 1. Domain Definition

| Attribute | Finding |
|-----------|---------|
| **Actor** | Collector (`RECORD_COLLECTIONS`) |
| **Trigger** | End-of-day cash count submission |
| **Inputs** | Collector ID, date, physical cash (pesewas) |
| **Expected** | Sum of weekly payment amounts for loans due on date |
| **Actual** | Sum of confirmed payments recorded by collector on date |
| **Variance** | Physical cash ÔêÆ actual (system payments) |
| **Flag** | Variance > 10% of expected (`RECONCILIATION_VARIANCE_THRESHOLD_PERCENT`) |
| **Lock** | One submission per collector + date (422 on duplicate) |

### Business logic

```13:26:apps/frontend/src/utils/reconciliation-summary.ts
export function buildReconciliationTotals(
  collectorId: string,
  date: string,
  loans: CollectorDashboardLoanInput[],
  payments: CollectorDashboardPaymentInput[],
): ReconciliationTotals {
  const dueLoans = loans.filter((loan) => isLoanDueOnDate(loan.paymentDay, date));
  const expectedPesewas = dueLoans.reduce((total, loan) => total + loan.weeklyPaymentPesewas, 0);
  const actualPesewas = payments
    .filter((payment) => payment.collectorId === collectorId && payment.paymentDate === date)
    .reduce((total, payment) => total + payment.amountPesewas, 0);

  return { expectedPesewas, actualPesewas };
}
```

## 4. Cross-Domain Relationships

### Payments (direct)

Reconciliation **actual** totals derive from `payments` filtered by `collectorId` + `paymentDate`. Backend `paymentService.recordPayment` writes CONFIRMED payments with ledger REPAYMENT entries ÔÇö **no reconciliation hook**.

### Ledger (none)

Reconciliation does not read or write `ledger_entries`. Reversal MVP (3C.1) adds REVERSAL entries ÔÇö **reconciliation totals do not account for reversed payments** in frontend logic today.

### Adjustments (alignment gap)

`P14.3B-adjustment-architecture-review.md`: approved adjustments change balances; reconciliation compares cash vs payments ÔÇö **no adjustment visibility in reconcile flow**.

### Pools (separate concept)

`verify:pools` checks pool balance integrity (`all pools reconcile`). Collector cash reconciliation is **orthogonal** ÔÇö payment service does not write REPAYMENT pool allocations per `P14.3B-pool-architecture-review.md`.

### Expenses (not included)

`buildReconciliationTotals` does not include collector expenses. Expenses table exists elsewhere; not wired to reconcile UI.
