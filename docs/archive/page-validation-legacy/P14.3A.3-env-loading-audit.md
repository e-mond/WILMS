## Implementation

**File:** `apps/backend/src/config/load-env.ts`

Load order (later files fill only **unset** keys):

1. `{monorepo}/.env`
2. Development: `apps/backend/.env`, `apps/backend/.env.local`, `{monorepo}/.env.local`
3. Production: `apps/backend/.env.production`, `{monorepo}/.env.production`

**Entry points importing load-env:**

| File | Purpose |
|------|---------|
| `src/index.ts` | API server startup |
| `src/db/seed/run-seed.ts` | Database seed |
| `src/verification/run-all.ts` | Financial harness |
| `drizzle.config.ts` | Migrations |

`loadEnvironment()` runs automatically on module import.

## Frontend Loading

Next.js automatically loads (in order):

- `.env`
- `.env.local`
- `.env.development` / `.env.production`
- `.env.development.local` / `.env.production.local`

No code changes required. Documented via `apps/frontend/.env*.example`.
