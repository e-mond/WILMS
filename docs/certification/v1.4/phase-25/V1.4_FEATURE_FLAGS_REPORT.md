# V1.4 Feature Flags Report

**Date:** 18 July 2026  
**Module:** `apps/backend/src/config/feature-flags.ts`  
**Tests:** `src/tests/platform/feature-flags.test.ts`

---

## Summary

Phase 25 introduces server-controlled feature flags backed by `WILMS_FLAG_*` environment variables with safe defaults. Flags gate platform foundation capabilities without client-side authorization. **`glDualWrite` is always false by default** — stub for v1.5 GL work.

---

## Flag catalog

| Flag | Env variable | Default (non-prod) | Default (production) | Purpose |
|------|--------------|--------------------|-----------------------|---------|
| `durableQueues` | `WILMS_FLAG_DURABLE_QUEUES` | `true` if `REDIS_URL` set | Same | Prefer BullMQ when Redis available |
| `requireIdempotency` | `WILMS_FLAG_REQUIRE_IDEMPOTENCY` | `false` | **`true`** | Require `Idempotency-Key` on money POSTs |
| `cursorPagination` | `WILMS_FLAG_CURSOR_PAGINATION` | `true` | `true` | Enable cursor pagination helpers |
| `outboxEnabled` | `WILMS_FLAG_OUTBOX` | `true` | `true` | Write `domain_outbox` rows |
| `outboxDelivery` | `WILMS_FLAG_OUTBOX_DELIVERY` | `true` if `REDIS_URL` | Same | Deliver outbox via queue |
| `tracingEnabled` | `WILMS_FLAG_TRACING` | `true` | `true` | Emit span logs |
| `glDualWrite` | `WILMS_FLAG_GL_DUAL_WRITE` | **`false`** | **`false`** | GL dual-write stub (v1.5) |

### Parsing rules

Truthy: `1`, `true`, `yes`, `on` (case-insensitive)  
Falsy: `0`, `false`, `no`, `off`  
Unset / empty: use default column above

---

## API surface

| Consumer | Exposure |
|----------|----------|
| Queue layer | `durableQueues` |
| Idempotency middleware | `requireIdempotency` |
| Cursor routes | `cursorPagination` (reserved; borrowers always opt-in via query) |
| Outbox writer/publisher | `outboxEnabled`, `outboxDelivery` |
| Tracing | `tracingEnabled` |
| Ops status | **All flags** returned in `featureFlags` object |

No REST endpoint allows clients to toggle flags — env/ops only.
