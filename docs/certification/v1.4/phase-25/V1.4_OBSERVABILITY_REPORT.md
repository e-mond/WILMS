# V1.4 Observability Report

**Date:** 18 July 2026  
**Scope:** Structured tracing spans, request correlation, Prometheus queue metrics, ops feature flag exposure

---

## Summary

Phase 25 adds lightweight OpenTelemetry-**compatible span logging** (JSON, no collector required), extends Prometheus metrics with queue gauges, and surfaces resolved feature flags on the ops status report. Request ID propagation predates Phase 25 and remains the trace correlation anchor.

## Operational guidance

1. Scrape `/ops/metrics` with existing Prometheus config.
2. Alert when `wilms_queue_bullmq=0` but `REDIS_URL` is set (unexpected fallback).
3. Correlate logs via `traceId` / `X-Request-Id` in log aggregation.
4. Do not enable OTLP until exporter is implemented (future phase).

---

## Conclusion

Observability **foundation** is in place: correlation IDs, span logs, queue Prometheus gauges, and flag visibility. **Enterprise SRE bar** (collector export, dashboards, prod alerts) is **not certified** in Phase 25.
