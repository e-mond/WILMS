# V1.4 Performance Report — Phase 25

**Date:** 18 July 2026  
**Scope:** Index additions, cursor pagination, queue architecture — **no load test executed**

---

## Summary

Phase 25 improves **list scalability foundations** via keyset pagination (borrowers) and composite indexes on high-volume tables. BullMQ enables **async offload** of mail/SMS/export/outbox work. **No production-scale load test or Neon `EXPLAIN` was run** in this certification.

---

## Delivered performance artifacts

### 1. Cursor / keyset pagination (borrowers)

| Aspect | Detail |
|--------|--------|
| Pattern | Keyset on `(registeredAt DESC, id DESC)` |
| Page size | Default 50, max 200 |
| vs offset | Avoids `OFFSET` scan degradation on deep pages |
| API opt-in | `pagination=cursor` — backward compatible |

**Expected benefit:** O(page size) per request vs O(offset + limit) for large borrower registries.

### 2. Database indexes (migration 0028)

| Index | Table | Columns | Intended use |
|-------|-------|---------|--------------|
| `borrowers_created_id_idx` | borrowers | `(created_at, id)` | Future cursor lists |
| `payments_created_id_idx` | payments | `(created_at, id)` | Payment history pages |
| `expenses_created_id_idx` | expenses | `(created_at, id)` | Expense list pages |

Prior work: `0027_hot_query_indexes` — dashboard/KPI hot paths.

### 3. Async job queue

| Workload | Before v1.4 | After v1.4 (with Redis) |
|----------|-------------|-------------------------|
| Mail send | Inline / fire-and-forget | BullMQ with retry |
| SMS send | Inline | BullMQ with retry |
| Outbox delivery | N/A | Async via `outbox` queue |
| Export jobs | Inline logging | Queue hook (handler logs) |

**Without Redis:** In-process — no latency improvement, no durability.
