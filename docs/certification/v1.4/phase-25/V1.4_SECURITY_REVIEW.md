# V1.4 Security Review — Phase 25 Platform Surfaces

**Date:** 18 July 2026  
**Scope:** New/modified platform infrastructure (queues, flags, outbox, idempotency, ops)  
**Production certification:** **Not claimed** — software review only

---

## Executive assessment

Phase 25 changes are **additive** and follow existing WILMS security patterns (server-side enforcement, no client flag mutation, structured logging with redaction). No new critical vulnerabilities identified in code review. Operator-dependent items (Redis ACLs, prod flag audit) remain open.

---

## Surface-by-surface review

### 1. Redis / BullMQ

| Control | Status | Notes |
|---------|--------|-------|
| Redis URL from env only | ✓ | `REDIS_URL` / `WILMS_REDIS_URL` |
| No Redis creds in logs | ✓ | Connection errors log message only |
| Fallback on failure | ✓ | Degrades to in-process — availability over durability |
| TLS to Redis | **Operator** | Depends on provider URL (`rediss://`) |
| Network isolation | **Not verified** | Railway/private network TBD |

**Risk:** Unauthenticated Redis in shared network → job injection. **Mitigation:** Provider ACL, private networking, strong password.

### 2. Idempotency

| Control | Status | Notes |
|---------|--------|-------|
| Key scoped to actor + scope | ✓ | Unique index prevents cross-user replay |
| Payload hash conflict → 409 | ✓ | Prevents semantic replay attacks |

**Risk:** Client omits key in prod → 400. **Mitigation:** Frontend/offline queue must emit keys before enforce rollout.

## Recommendations (pre-prod deploy)

1. Provision Redis with TLS and ACL; never expose publicly.
2. Confirm frontend emits `Idempotency-Key` before enabling prod (default on).
3. Complete Neon restore drill; secure evidence artifacts.
4. Review outbox payload schema before wiring money mutations.
5. Keep `WILMS_FLAG_GL_DUAL_WRITE=false` until v1.5 GL consumer exists.

---

## Conclusion

Phase 25 security posture is **acceptable for merge to main** as platform foundation code. **Production security sign-off** requires operator completion of Redis hardening, client idempotency compliance, and restore drill — not claimed in this phase.
