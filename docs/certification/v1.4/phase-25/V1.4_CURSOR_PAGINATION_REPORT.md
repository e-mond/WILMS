# V1.4 Cursor Pagination Report

**Date:** 18 July 2026  
**Scope:** Keyset pagination helpers + borrowers list API opt-in

---

## Summary

Phase 25 introduces opaque cursor (keyset) pagination utilities and wires **borrowers** `GET /borrowers` to support `pagination=cursor`. Offset pagination remains fully supported. Migration `0028` adds `(created_at, id)` indexes on borrowers, payments, and expenses for future list endpoints.

---

## Core module

**Path:** `apps/backend/src/http/cursor-pagination.ts`

| Export | Purpose |
|--------|---------|
| `encodeCursor` / `decodeCursor` | Base64url JSON `{ v, id }` |
| `parseCursorListQuery` | Activates on `pagination=cursor` or `cursor` param |
| `buildCursorPage` | Builds `items`, `nextCursor`, `prevCursor`, `limit` |
| `DEFAULT_CURSOR_PAGE_SIZE` | 50 |
| `MAX_CURSOR_PAGE_SIZE` | 200 |

Activation rules:

- Client sends `pagination=cursor`, or
- Client sends `cursor` query parameter

Otherwise returns `null` → route falls back to offset or full list.

## Borrowers API

**Route:** `GET /borrowers` (`apps/backend/src/modules/borrowers/routes.ts`)

| Mode | Query | Response shape |
|------|-------|----------------|
| Cursor | `pagination=cursor&limit=50&cursor=<opaque>` | `{ items, nextCursor, prevCursor, limit }` via `sendData` |
| Offset | `page` / `pageSize` (existing) | Paginated envelope via `sendPaginatedData` |
| Full | No pagination params | Full list (legacy) |

**Keyset columns:** `registeredAt` (sort value `v`) + `id` (tie-breaker)  
**Order:** Newest registered first (`DESC registeredAt, DESC id`)

**Repository:** `apps/backend/src/repositories/borrower.repository.ts`

```sql
-- Conceptual keyset predicate
WHERE (registered_at < :cursorDate)
   OR (registered_at = :cursorDate AND id < :cursorId)
ORDER BY registered_at DESC, id DESC
LIMIT :limit
```

## Database indexes (migration 0028)

| Index | Table | Columns |
|-------|-------|---------|
| `borrowers_created_id_idx` | `borrowers` | `(created_at, id)` |
| `payments_created_id_idx` | `payments` | `(created_at, id)` |
| `expenses_created_id_idx` | `expenses` | `(created_at, id)` |

**Note:** Borrowers cursor uses `registeredAt` at the application layer; index on `created_at` supports planned expansion to payments/expenses lists. Prior migration `0027_hot_query_indexes` added complementary hot-query indexes.
