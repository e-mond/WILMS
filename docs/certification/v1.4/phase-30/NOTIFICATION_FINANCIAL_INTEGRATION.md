# Notification Financial Integration

**Version:** 1.4.2 | **Phase:** 30

## Integration points

| Event | Hook location | Transaction coupling |
|-------|---------------|---------------------|
| Payment confirmed | `modules/payments/service.ts` after `runInTransaction` | **Decoupled** — async `void emit...` |
| Payment due soon | `payment-scheduler.service.ts` | Read-only schedule scan |
| Payment missed | Scheduler after `applyMissedWeekMarking` | Schedule status update then notify |
| Payment reversal | `payment-reversal.service.ts` (existing) | Unchanged |

## Schedule authority

Due dates originate from `loan_schedules` rows generated by `generateLoanScheduleWeeks` with holiday shift at disbursement. Missed determination uses `applyMissedWeekMarking` with `latePaymentGraceDays` from system settings.

## Idempotency

Payment API idempotency (`runWithIdempotency`) is separate from notification dedupe. Notification dedupe keys prevent duplicate messages if payment or scheduler retries.

## Ledger integrity

Notification failure does not roll back payments. No money mutation in notification code paths.

## Operational ledger note

WILMS remains an operational sub-ledger — not statutory double-entry GL.
