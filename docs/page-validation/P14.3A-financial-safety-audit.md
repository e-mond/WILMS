## Monetary Precision

| Field class | Storage | Evidence |
|-------------|---------|----------|
| Principal, balance, installment, ledger | `NUMERIC(15,2)` | `loans.ts`, `ledger-entries.ts`, `loan-schedules.ts` |
| Interest/penalty intermediate | `NUMERIC(15,6)` | `loans.interest_amount`, `penalty_amount` |
| API boundary | Integer pesewas | `domain/money.ts` |

Currency code `GHS` defaulted on all financial tables.

## Transaction Atomicity

| Operation | All-or-nothing bundle |
|-----------|----------------------|
| Loan create | loan + schedules + ledger |
| Disburse | lifecycle ×2 + disbursement + ledger |
| Repayment | payment + schedule + loan balance + ledger |

**Evidence:** `runInTransaction()` in `db/client.ts`; service layer usage.

## Idempotency

| Scope | Unique key | Retention |
|-------|------------|-----------|
| LOAN_DISBURSE | `(scope, actor_user_id, idempotency_key)` | 24h |
| PAYMENT_POST | same | 24h |

Completed replay returns stored response; in-flight duplicate → 409; failed attempt deletes in-progress key for retry.

**Evidence:** `idempotency-keys.ts` schema; `run-with-idempotency.ts`; `idempotency.repository.ts`.

## Rounding

Weekly installment: `Math.floor(amountPesewas / durationWeeks)` — no remainder absorption.

**Evidence:** `domain/loan/calculations.ts`.

## Audit Coverage

All financial mutations call `appendAuditEntry()`: `loan.created`, `loan.approved`, `loan.rejected`, `loan.disbursed`, `payment.recorded`.
