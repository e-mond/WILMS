## Root cause

| Layer | Finding |
|-------|---------|
| Railway `/auth/login` | **200** — credentials valid |
| BFF `POST /api/auth/login` (before fix) | **401** — all roles |
| CSRF on BFF | **403** without token — correct |

**Classification:** **Deployment / Application** — BFF `authenticateCredentials()` used `apiClient.post('/auth/login')`, routing through `/api/wilms` proxy. Server-side fetch does not forward CSRF cookies → proxy rejected → BFF returned 401.

**Fix:** [`apps/frontend/src/lib/auth/authenticate.ts`](../../apps/frontend/src/lib/auth/authenticate.ts) — call `WILMS_API_UPSTREAM/auth/login` directly from BFF login handler.

**Vercel deploy:** `dpl_62q7Q57V9y9jPkVzcQn2KeUVZFy4` → `https://wilms.vercel.app`
