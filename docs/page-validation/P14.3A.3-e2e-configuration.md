## Changes

**File:** `apps/frontend/playwright.config.ts`

All URLs and ports are now environment-driven.

| Variable | Purpose | Default |
|----------|---------|---------|
| `PLAYWRIGHT_BASE_URL` | Full E2E base URL | `http://{host}:{port}` |
| `PLAYWRIGHT_HOST` | Bind hostname | `127.0.0.1` |
| `PLAYWRIGHT_PORT` | E2E dev server port | — |
| `FRONTEND_PORT` | Alias for port | — |
| `E2E_PORT` | Legacy alias | `3000` |
| `E2E_HOST` | Legacy alias | `127.0.0.1` |
| `CI` | `reuseExistingServer: false` when set | — |
| `NEXT_PUBLIC_APP_LOCK_IDLE_MS` | Passed to webServer env | `3600000` |
| `NEXT_PUBLIC_API_BASE_URL` | Passed to webServer env | `""` |

### Resolution order (port)

```text
PLAYWRIGHT_PORT → FRONTEND_PORT → E2E_PORT → '3000'
```

---

## Environment Setups

### Local development

```env
# apps/frontend/.env.local
PLAYWRIGHT_PORT=3001
PLAYWRIGHT_BASE_URL=http://127.0.0.1:3001
E2E_PORT=3001
```

Use port **3001** when `npm run dev` occupies **3000**.

### CI

```bash
CI=1 npm run test:e2e
```

- `reuseExistingServer: false` — Playwright starts its own server
- Retries: 2

### Staging / production E2E (future)

```env
PLAYWRIGHT_BASE_URL=https://staging.wilms.example
# No webServer — use external URL testing project
```

## Prior Issue (P14.3A.2)

E2E failed when frontend dev server already bound `:3001`:

```text
http://127.0.0.1:3001/login is already used
```

**Remediation:** Stop dev servers before `CI=1 npm run test:e2e`, or use distinct `PLAYWRIGHT_PORT`.

## webServer Command

```text
npm run dev -- --hostname {PLAYWRIGHT_HOST} --port {PORT}
```

Waits for `{PLAYWRIGHT_BASE_URL}/login`.
