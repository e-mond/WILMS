## Mode resolution (authoritative)

[`apps/frontend/next.config.mjs`](../../apps/frontend/next.config.mjs) `shouldUseMockServices()` + [`types.ts`](../../apps/frontend/src/data-provider/types.ts) `resolveDataProviderMode()`.

| Switch | Effect |
|--------|--------|
| `NEXT_PUBLIC_USE_MOCK=false` + API URL | **Live** (webpack → `index.production.ts`) |
| Dev default | **Mock** |
| `NEXT_PUBLIC_DEMO_MODE` / `FORCE` / `API_DISABLED` | **Mock** |

**Restart required** to change mode (webpack alias at compile time). No hot env toggle.

## APP_MODE design recommendation (no implementation)

| Mode | Maps from | Use case |
|------|-----------|----------|
| `mock` | DEMO/FORCE/API_DISABLED or dev default | UI demos, E2E |
| `local` | `USE_MOCK=false` + local BFF + backend | Developer live testing |
| `staging` | production build + staging URL + Cloudinary | Pre-prod cert |
| `production` | NODE_ENV=production + secrets | Deploy |

**Recommendation:** **Defer** implementation to post-v1 (effort M, risk medium). Document current flags in README (done in P14.3B alignment).

**Backward compatibility:** Shim must read legacy `NEXT_PUBLIC_*` flags if `APP_MODE` introduced.
