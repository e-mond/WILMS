## Purpose

Establish measured latency and query-pattern baseline **before** P14.3B feature work and Phase 6 backend optimization. No optimizations were applied in this phase.

| Item | Value |

## Bottleneck Summary (pre-optimization)

| ID | Area | Root cause | Phase 6 target |
|----|------|------------|----------------|
| PERF-B01 | Loan list | No SQL `WHERE external_status`; full scan + JS filter | Push filter to SQL + index |
| PERF-B02 | Schedule | `applyMissedWeekMarking` write-on-read; multi-query | Bulk update / separate mutation path |
| PERF-B03 | Portfolio | 2× borrower fetch per loan | JOIN or batch borrower load |
| PERF-B04 | Payments/reports | Full table scans | Indexed filters (`payment_date`, `collector_user_id`) |
| PERF-B05 | Neon RTT | ~250ms floor per round trip | Batch queries; reduce round trips |
