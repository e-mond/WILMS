## Root causes (all issues, not only Output Directory)

### Issue 1 — Primary: Root Directory / output path mismatch

- **Symptom:** `.next` was not found  
- **Cause:** Build output at `apps/frontend/.next`, Vercel expected `./.next` at project root  
- **Fix:** Set Vercel **Root Directory** = `apps/frontend` + `apps/frontend/vercel.json`

### Issue 2 — Secondary: Monorepo install scope

- **Symptom:** Potential missing workspace packages during install  
- **Cause:** Installing only inside `apps/frontend` without hoisted `node_modules` from root  
- **Fix:** `installCommand: "cd ../.. && npm ci"` in `vercel.json`

### Issue 3 — Architectural: Backend not on Vercel

- **Symptom:** Deployed frontend with live API mode returns 500/502 from BFF  
- **Cause:** `WILMS_API_UPSTREAM` pointed to `127.0.0.1:4000` or unset  
- **Fix:** Deploy Express to Railway; set `WILMS_API_UPSTREAM=https://<railway-api-host>`  
- **Code hardening (P14.5C):** BFF throws if `WILMS_API_UPSTREAM` missing in production (`route.ts`)

### Issue 4 — CLI / credentials (P14.5B)

- **Symptom:** Deploy failed after CLI upgrade  
- **Cause:** Invalid Vercel token — not a build configuration issue  
- **Fix:** `vercel login`

### Issue 5 — Environment: mock mode without API URL

- **Symptom:** App runs but uses mock data in production  
- **Cause:** `NEXT_PUBLIC_API_BASE_URL` unset → `shouldUseMockServices()` returns true  
- **Fix:** Set `NEXT_PUBLIC_API_BASE_URL` + `NEXT_PUBLIC_USE_MOCK=false`

## Conclusion

The `.next` error is **one symptom** of monorepo misconfiguration. Full production requires **three** fixes: correct Vercel root, monorepo install/build commands, and **separate backend hosting** with `WILMS_API_UPSTREAM` configured.
