## Required persistent model (blueprint summary)

Single **`uploads`** table (confirmed) with:

- `id`, `purpose`, `file_name`, `mime_type`, `size_bytes`, `url`, `entity_id`, `uploaded_at`, `storage_path`
- FK references from `borrowers` (multiple upload ID columns from payload)
- FK from `expenses.receipt_upload_id`

**Join pattern:** Entity tables store upload UUID; binary in object storage or filesystem path in `storage_path`.
