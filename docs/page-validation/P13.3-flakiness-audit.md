# P13.3 Flakiness Audit

**Date:** 2026-06-09  
**Baseline flaky (P13.2 r2):** 10 tests  
**Final (P13.3 r2):** **0 flaky**

## Root Causes Addressed

### 1. App lock idle during long sequential suite

**Evidence:** `P13.3-e2e-run-r1.log` — app-lock dialog intercepted clicks in `responsive-breakpoints.spec.ts` @ mobile-320 and shell tests.  
**Cause:** `playwright.config.ts` set `NEXT_PUBLIC_APP_LOCK_IDLE_MS: '2000'` for all tests.  
**Fix:**
- Default E2E idle → `3600000` (`playwright.config.ts`)
- Runtime override in `src/constants/app-lock.ts` via `window.__WILMS_E2E_APP_LOCK_IDLE_MS`
- `app-lock.spec.ts` `beforeEach` sets `2000` only for that file

### 2. Axe scan timing / hydration

**Fix:** `waitForAxeReady()` waits for `html[data-theme-store-ready="true"]`; `accessibility.spec.ts` `test.setTimeout(60_000)`.

### 3. Navigation load stalls

**Fix:** Super-admin axe and shell navigations use `{ waitUntil: 'domcontentloaded' }` where `load` hung (risk-flags, collectors).

### 4. Live badge contrast sampling during pulse

**Fix:** Pulse animation on dot only — static text colors for axe (`AppNavbar.tsx`).
