## Summary

Implements P14.3B Phase 2 financial adjustments with approval workflow, ledger `ADJUSTMENT` entries, append-only history (before/after/delta), audit wiring, and `verify:adjustments` harness (10/10).

Includes Phase 1 certification docs and engineering governance (`CONTRIBUTING.md`, branching strategy).

## Scope

**In scope**

- Schema: `financial_adjustments`, `adjustment_reasons`, `adjustment_history`
- API: pending list, CRUD list/detail, create, approve, reject
- Atomic approve transaction (ledger + balance + pool + history)
- Performance: SQL loan status filter, batch portfolio borrower lookup (−57% portfolio P95)

**Out of scope**

- Reversals (Phase 3)
- Reconciliation (Phase 4)
- Dedicated write-off service (Phase 5)
