# P14.3A — Loan Lifecycle Report

**Phase:** P14.3A  
**Date:** 2026-06-09  
**Status:** Complete

---

## Summary

Server-side loan lifecycle is implemented with internal `loan_lifecycle_status` mapped to frontend `LOAN_STATUS` via persisted `external_status`.

---

## Lifecycle States

| Internal status | Allowed next states | API `status` |
|-----------------|---------------------|--------------|
| DRAFT | PENDING_APPROVAL | PENDING_DISBURSEMENT |
| PENDING_APPROVAL | APPROVED, REJECTED | PENDING_DISBURSEMENT |
| APPROVED | PENDING_DISBURSEMENT | PENDING_DISBURSEMENT |
| REJECTED | — | WRITTEN_OFF |
| PENDING_DISBURSEMENT | DISBURSED | PENDING_DISBURSEMENT |
| DISBURSED | ACTIVE | ACTIVE |
| ACTIVE | COMPLETED, DEFAULTED | ACTIVE |
| COMPLETED | — | COMPLETED |
| DEFAULTED | ACTIVE, WRITTEN_OFF | DEFAULTED |
| WRITTEN_OFF | — | WRITTEN_OFF |

**Evidence:** `apps/backend/src/domain/loan/lifecycle.ts` — `TRANSITIONS` map and `assertLifecycleTransition()`.

---

## Service Operations

| Operation | Endpoint | Lifecycle effect | Audit |
|-----------|----------|------------------|-------|
| Create loan | `POST /loans` | → PENDING_DISBURSEMENT + schedule | `loan.created` |
| Approve | `PATCH /loans/:id/approve` | → PENDING_DISBURSEMENT (idempotent if already pending) | `loan.approved` |
| Reject | `PATCH /loans/:id/reject` | → REJECTED | `loan.rejected` |
| Disburse | `POST /loans/:id/disburse` | PENDING_DISBURSEMENT → DISBURSED → ACTIVE | `loan.disbursed` |

**Evidence:** `apps/backend/src/modules/loans/service.ts` — `createLoan`, `approveLoan`, `rejectLoan`, `disburseLoan`.

---

## Transaction Boundaries

| Operation | Atomic bundle |
|-----------|---------------|
| Create | loan + schedule + ledger (INTEREST_CHARGE placeholder) |
| Approve | lifecycle update + ledger |
| Disburse | lifecycle (×2) + disbursement + ledger (LOAN_DISBURSEMENT) |

**Evidence:** `runInTransaction()` calls in `service.ts`.

---

## Soft Delete

Loans table includes `deleted_at`, `deleted_by_user_id`. No hard-delete service methods exposed.

**Evidence:** `apps/backend/src/db/schema/loans.ts`; `findLoanById` filters `deletedAt IS NULL`.

---

## Invalid Transition Rejection

Invalid transitions throw `VALIDATION:Invalid lifecycle transition …` → HTTP 422.

**Evidence:** `assertLifecycleTransition()` in `lifecycle.ts`; mapped in `modules/loans/routes.ts`.
