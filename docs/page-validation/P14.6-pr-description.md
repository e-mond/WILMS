## Summary

Release **v0.2.2** — offline sync, CI/CD hardening, and UX improvements (P14.6).

- Primary URL: **https://wilms.vercel.app**
- API: **https://wilms-production.up.railway.app**
- Branch: `release/p14.6-cd-sync-ux`
- Local validation: type-check, lint, build, bundle budgets, **205/205** frontend + **12/12** backend tests

## Changes

### Offline Sync
- Migration `0007_offline_sync` — operations queue + conflict review tables
- `POST /sync/offline/batch` with idempotency (`actor_user_id` + `idempotency_key`)
- Financial ops (`RECORD_PAYMENT`) queued for approver review — never auto-applied
- `GET /sync/conflicts`, `POST /sync/conflicts/:id/approve|reject`
- Frontend offline payment replay routes through sync API

### CI/CD
- Enhanced `ci.yml`: bundle + perf budget checks, `npm audit`, gitleaks secret scan
- New `deploy-staging.yml` (gated by `ENABLE_STAGING_DEPLOY`)
- `deploy-production.yml`: manual `workflow_dispatch`, health baseline, version check, smoke tests

### UX & Performance
- Route-level `loading.tsx` for collector and approver shells
- Skip-to-content link (`SkipToContent` in root layout)
- Bundle budgets enforced: **168.5 KB** gzip JS (budget 350 KB), shared first load **87.8 KB**

### Security & Hardening
- Production mock-flag guard (`assertProductionMockDisabled`) — API exits on mock env vars
- `apiClient` no longer redirects to session-expired on 401 while on `/login`
- PWA install banner: `preventDefault` only when custom UI will show
- CSRF (P14.5G synchronizer token) — unchanged

### Documentation
- P14.6 audit suite: sync architecture, security scan, accessibility, performance, validation
- Migration rollback guide for `0007_offline_sync`
- Neon PITR backup procedure documented

## Breaking changes

None. Offline sync is additive; existing payment flows unchanged until conflict approval.
