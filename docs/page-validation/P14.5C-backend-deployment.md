# P14.5C — Backend Deployment Guide

**Platform:** Railway (recommended)  
**Service:** `@wilms/api` (`apps/backend`)

---

## Prerequisites

- Neon `DATABASE_URL` (production branch)
- Cloudinary credentials (`UPLOAD_PROVIDER=cloudinary`)
- `WILMS_SESSION_SECRET` (≥32 random bytes)
- `WILMS_CORS_ORIGIN` = production frontend URL

---

## Railway setup

1. Create new project → **Deploy from GitHub** → `e-mond/WILMS`
2. Add service for backend:
   - **Root Directory:** repository root (or configure via `apps/backend/railway.toml`)
   - **Start Command:** `npm run start -w @wilms/api`
   - **Build Command:** `npm ci`
3. Set environment variables (see `P14.5C-production-environment.md`)
4. Enable **health check:** path `/health`, port from `PORT` env
5. Run migrations once: `npm run db:migrate -w @wilms/api` (Railway one-off job or CI)

---

## `apps/backend/railway.toml`

```toml
[build]
builder = "nixpacks"
buildCommand = "cd ../.. && npm ci"

[deploy]
startCommand = "cd ../.. && npm run start -w @wilms/api"
healthcheckPath = "/health"
healthcheckTimeout = 60
```

---

## Runtime characteristics

| Item | Detail |
|------|--------|
| Start command | `tsx src/index.ts` (`package.json` `start`) |
| Port | `process.env.PORT` (Railway injects) — `env.ts` L31 |
| Host | `0.0.0.0` in production (P14.5C default) |
| Trust proxy | `true` in production (`env.ts` L27) |
| Graceful shutdown | SIGTERM/SIGINT handlers (`index.ts`) |

---

## Post-deploy verification

```bash
curl -fsS "https://<api-host>/health" | jq .
```

Expected: `status: "ok"`, `database.connected: true`, `uploads.valid: true`.

---

## Why not serverless

| Blocker | File |
|---------|------|
| `app.listen()` persistent server | `index.ts` |
| In-memory rate limit store | `login-rate-limit.ts` |
| Local filesystem uploads (dev) | `local-provider.ts` |
| No compiled `dist/` or handler export | `package.json`, `tsconfig` |

Refactoring to serverless is **out of P14.5C scope**.

---

## Alternatives

- **Render:** use `render.yaml` `wilms-api` service definition
- **Fly.io:** requires new `Dockerfile` (not provided in P14.5C)
