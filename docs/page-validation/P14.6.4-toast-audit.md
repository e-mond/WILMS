## Coverage gaps (no toast on success/error)

These mutation-heavy surfaces rely on inline UI state or silent success — **no `useToast` or `mutation-feedback` import found**:

| Domain | Representative files | Gap |
|--------|---------------------|-----|
| Payment collection | `PaymentEntryPanel.tsx`, `GroupCollectionSheet.tsx`, `CollectorDashboardPanel.tsx` | Payment success/failure not uniformly toasted |
| Reconciliation | `ReconciliationForm.tsx` | Submit feedback via form state only |
| Adjustments | `AdjustmentsPanel.tsx` | No toast on approve/reject |
| Overpayment review | `OverpaymentReviewPanel.tsx` | No toast hooks |
| Loan management | `LoanDetailPanel.tsx`, `BorrowerLoanDetailPanel.tsx` | No toast hooks |
| Reports (export aside) | Report panels use `ExportCsvButton` (has toast) for CSV only | Report load errors → spinner/empty state |
| Notifications inbox | `NotificationInboxPanel.tsx` | Mark-read actions without toast |

**Risk:** Users may miss confirmation on critical financial actions (payments, reconciliation submit).

---

## Variant usage summary

| Variant | Primary use |
|---------|-------------|
| `SUCCESS` | Settings save, group updates, export complete, risk flag resolved |
| `ERROR` | `notifyMutationError` on wizard failures |
| `WARNING` | Risk flag escalation, sync incomplete |
| `INFO` | Officer assignment, back online |
| `OFFLINE` | Persistent offline banner (`durationMs: 0`) |
| `SYNC` | Offline queue upload complete |

---

## Recommendations (v0.3.0 backlog)

1. Standardize payment + reconciliation flows on `notifyMutationSuccess` / `notifyMutationError`.
2. Add error toasts to TanStack Query `onError` for new P14.6.4 surfaces (dashboard, search) where only loading spinners exist.
3. Keep E2E coverage via `apps/frontend/e2e/toast-feedback.spec.ts` when adding toasts.

## Verdict

**PASS (infrastructure)** / **GAPS (coverage)** — Toast system is production-ready; financial workflow panels need consistent feedback in a future UX pass.
