## Observations

1. **Stabilisation:** Both endpoints show elevated first request, then settle. API warm mean ~813ms; BFF warm mean ~1453ms (extra Vercel hop).
2. **Railway sleep:** No evidence of full sleep-to-wake (would often be 10s+). Behaviour consistent with **cold-ish start / regional latency** rather than hobby-tier full sleep during this window.
3. **Vercel:** Next.js App Router uses **serverless functions** for `/api/wilms/*` — cold starts add overhead on first BFF request.
4. **Stakeholder expectation:** Warm API health ~0.6–1.1s; warm BFF health ~0.9–2.2s from US-East probe.

---

## Recommendations

| Issue | Mitigation |
|-------|------------|
| BFF cold start >3s | Vercel production alias + redeploy warms edge; consider Railway region closer to Vercel `iad1` |
| API first-hit ~2s | Railway paid plan / min instances if latency SLA required |
| Ongoing monitoring | Add external uptime check on `/health` and `/api/wilms/health` |

**Note:** Login endpoint perf not measured in this run (auth POST requires body); recommend separate 10-sample run after credential hardening.
