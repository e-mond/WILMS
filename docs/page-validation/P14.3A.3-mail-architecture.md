# P14.3A.3 — Mail Architecture

**Phase:** P14.3A.3 Phase 6  
**Date:** 2026-06-09  
**Status:** Complete (infrastructure only)

---

## Design

Provider-based mail layer. **No sending workflows** — adapters only.

```text
apps/backend/src/infrastructure/mail/
├── types.ts          # MailMessage, MailProvider interface
├── config.ts         # MAIL_PROVIDER, SMTP_*, RESEND_*
├── smtp-adapter.ts   # nodemailer transport
├── resend-adapter.ts # Resend REST API
└── index.ts          # getMailProvider()
```

---

## Providers

| Provider | `MAIL_PROVIDER` | Required env | Adapter |
|----------|-----------------|--------------|---------|
| None (default) | `none` | — | NoopMailProvider (throws on send) |
| SMTP | `smtp` | `SMTP_HOST`, `SMTP_USER`, `SMTP_PASSWORD` | `SmtpMailProvider` |
| Resend | `resend` | `RESEND_API_KEY` | `ResendMailProvider` |

### Usage (future workflows)

```ts
import { getMailProvider } from './infrastructure/mail/index.js';

const mail = getMailProvider();
if (mail.isConfigured()) {
  await mail.send({ to: 'user@example.com', subject: '...', text: '...' });
}
```
