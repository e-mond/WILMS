# P13.3 Accessibility Audit (WCAG 2.1 AA)

**Date:** 2026-06-09  
**Validation:** `accessibility.spec.ts` — **18/18 axe scans pass** (desktop + mobile) in `P13.3-e2e-run-r2.log`

## Method

- Parsed axe violations from `P13.2-e2e-run-v4-r2.log` and `P13.3-e2e-run-r1.log`.
- Fixed product markup/tokens only; no layout or copy redesigns.
- Added `e2e/helpers/accessibility.ts` `waitForAxeReady()` (theme store gate) and 60s test timeout for axe analyze stability.

## Violations Closed

### 1. `nested-interactive` — Data tables with row links

**Evidence:** Groups page `<tr role="button">` containing `<Link href="/groups/...">` (`GroupsManagementPanel.tsx` L191–214).  
**Fix:** `DataTable.tsx` — removed `role="button"`, `tabIndex`, `aria-pressed`, keyboard handlers; row click skips nested interactive targets.

### 2. `scrollable-region-focusable`

| Element | File | Fix |
|---------|------|-----|
| Table wrapper | `DataTable.tsx` | `role="region"`, `aria-label`, `tabIndex={0}` |
| Loan schedule | `LoanScheduleTable.tsx` | Same pattern |
| Context aside | `AppAside.tsx` | `tabIndex={0}` on `#app-aside` |

### 3. `color-contrast`

| Surface | File | Fix |
|---------|------|-----|
| Live badge | `AppNavbar.tsx` | `bg-status-active-light text-status-active` + pulsing dot (`aria-hidden`) |
| Invite User / modal primary | `SettingsUsersSection.tsx`, `SettingsUserModal.tsx` | `text-white` on `bg-executive-gold` |
| Bottom nav inactive tabs | `ShellNavLink.tsx` `tabMode` | `text-text-muted` (was `text-text-tertiary`, 3.53:1 on white) |

### 4. `aria-required-parent` — mobile bottom navigation

**Evidence:** `<a role="tab">` without `tablist` parent (`OperationalMobileNavigation.tsx`, `ShellNavLink.tsx` L157–207).  
**Fix:** `<ul role="tablist">`, `<li role="presentation">`.
