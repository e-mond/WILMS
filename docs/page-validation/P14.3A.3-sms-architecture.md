# P14.3A.3 — SMS Architecture

**Phase:** P14.3A.3 Phase 7  
**Date:** 2026-06-09  
**Status:** Complete (infrastructure only)

---

## Design

Provider-based SMS layer. **No business workflows** — adapters only.

```text
apps/backend/src/infrastructure/sms/
├── types.ts            # SmsMessage, SmsProvider interface
├── config.ts           # SMS_PROVIDER, ARKESEL_*, TWILIO_*
├── arkesel-adapter.ts  # Arkesel REST API
├── twilio-adapter.ts   # Twilio Messages API
└── index.ts            # getSmsProvider()
```

---

## Providers

| Provider | `SMS_PROVIDER` | Required env | Adapter |
|----------|----------------|--------------|---------|
| None (default) | `none` | — | NoopSmsProvider |
| Arkesel | `arkesel` | `ARKESEL_API_KEY`, `ARKESEL_SENDER_ID` | `ArkeselSmsProvider` |
| Twilio | `twilio` | `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_PHONE_NUMBER` | `TwilioSmsProvider` |

### Usage (future workflows)

```ts
import { getSmsProvider } from './infrastructure/sms/index.js';

const sms = getSmsProvider();
if (sms.isConfigured()) {
  await sms.send({ to: '+233...', body: '...' });
}
```
