## Architecture Decisions

1. **Append-only accounting** — never delete/update ledger; compensating `REVERSAL` entries with FK to source ledger row.
2. **Dedicated `financial_reversals` + `reversal_history`** — do not reuse adjustment tables for execution semantics.
3. **Payment reversal MVP for 3C** — disbursement reversal deferred (B-ARCH-04); WRITE_OFF adjustment reversal blocked (B-ARCH-02).
4. **Mutual exclusion** — block reversal if approved `PAYMENT_CORRECTION` exists on same payment (B-ARCH-01).
5. **RBAC** — execute via `ACCESS_ADMIN_PORTAL`; read via `VIEW_FINANCIAL_REPORTS` (no new roles).
6. **Idempotency** — new scopes `REVERSAL_EXECUTE` + unique `(source_type, source_id)` constraint.
7. **Pool** — compensating allocation row with `reverses_allocation_id`; sub-step of payment/adjustment reversal only.
