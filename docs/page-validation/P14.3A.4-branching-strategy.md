## Branch Model (effective P14.3A.4)

```text
main                 # Production-ready; merge from release/* only
release/p14.3a       # P14.3A stabilization + certification (ACTIVE)
feature/p14-3b-*     # Future P14.3B domains (branch from release/p14.3a)
hotfix/*             # Production patches from main
```

**Note:** `develop` branch is documented in P14.3A.3 git strategy but **not yet created** in remote. Until `develop` exists, `release/p14.3a` is the integration branch for backend financial work.

## Workflow Rules

| Rule | Detail |
|------|--------|
| No direct P14.3B features on `main` | Use `feature/p14-3b-*` → PR → `release/p14.3a` |
| P14.3A fixes | Commit to `release/p14.3a`; cherry-pick to `main` when certified |
| Certification docs | Land on `release/p14.3a` under `docs/page-validation/P14.3A.4-*` |
| Secrets | Never commit `.env` |
| Merge to `main` | Requires partial cert gates + review |
