## Phase 1 — Variance Formula Alignment

### Problem

Frontend preview used `physical_cash − actual_collected`. Backend persists `physical_cash − expected_due` (primary variance v1).

### Evidence

| File | Line | Previous Formula | Backend Formula |
|------|------|------------------|-----------------|
| `apps/frontend/src/features/reconciliation/components/ReconciliationForm.tsx` | 59–61 | `physical − actualPesewas` | `physical − expectedPesewas` |
| `apps/frontend/src/utils/reconciliation-summary.ts` | 35–39 | Renamed to `calculatePrimaryVariancePesewas` | `apps/backend/src/domain/reconciliation/variance.ts` |
| `apps/frontend/src/services/mock/reconciliationService.mock.ts` | 84–87 | Updated to expected due | Same as backend |

### Fix

- Introduced `calculatePrimaryVariancePesewas(physicalCash, expectedDue)` matching backend.
- UI preview and mock submit now use `data.expectedPesewas`.
- Alert copy updated: "Cash matches expected collections".

### Tests

- `reconciliation-summary.test.ts`: zero, positive, negative variance cases.
- `reconciliationService.mock.test.ts`: zero-variance submit uses `physicalCashPesewas: 11000` (matches expected due on seed date).

**Exit criteria:** PASS — UI preview formula matches backend primary variance.

### Fix

### Tests

**Exit criteria:** PASS — invalid flagged submissions blocked client-side; rules match backend.

## Phase 3 — HTTP 403 Error Disambiguation

### Problem

All 403 responses displayed "Session expired".

### Fix

| Status | Code | User message |
|--------|------|--------------|
| 401 | `UNAUTHORIZED` | Your session has expired. Please sign in again. |
| 403 | `FORBIDDEN` | You do not have permission to perform this action. |

- `API_ERROR_CODE.FORBIDDEN` added in `types/api.ts`.
- `apiClient.ts` maps 401/403 separately; unauthorized handler triggers only on 401.
- `apiClient.test.ts` covers both paths.

**Exit criteria:** PASS — 401 and 403 produce distinct UI states.

## Phase 4 — Dev Environment Live API Enablement

### Problem

`npm run dev` always used mock services.

### Fix

Set in `.env.local` (documented in `.env.local.example`):

```bash
NEXT_PUBLIC_USE_MOCK=false
NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:3000/api/wilms
WILMS_API_UPSTREAM=http://127.0.0.1:4000
```

- `resolveDataProviderMode()` in `data-provider/types.ts` returns `api` when `NEXT_PUBLIC_USE_MOCK=false` and API URL is set.
- `next.config.mjs` webpack alias respects the same flag.
- Mock remains default when flag is unset.

### Tests

- `resolve-data-provider-mode.test.ts`: api mode with flag; mock default without.

**Exit criteria:** PASS — live backend reachable via env config only.
