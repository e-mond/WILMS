# P14.1C — Index Strategy

**Phase:** P14.1C (architecture only — no CREATE INDEX statements)  
**Date:** 2026-06-09  
**Governing:** ADR-001, P14.1C table architecture

Each index includes **why** it exists — tied to query patterns from services, conflict checks, and reports.

---

## `users`

| Index | Type | Columns | Why |
|-------|------|---------|-----|
| PK | Primary | id | UUID v7 lookup |
| UQ | Unique | email | Login — `LoginInput.email`, `checkPhone`-class uniqueness |
| IDX | B-tree | status, deleted_at | Active user lists — settings |
| IDX | B-tree | deleted_at | Soft-delete filter on all queries |

## `audit_entries`

| Index | Type | Columns | Why |
|-------|------|---------|-----|
| PK | Primary | id | — |
| IDX | B-tree (DESC) | created_at | Default list sort — `listRecentEntries` |
| IDX | B-tree | actor_id, created_at | Actor filter — `AuditListParams.actorId` |
| IDX | B-tree | action, created_at | Action filter |
| IDX | B-tree | target_entity_type, target_entity_id | Entity audit trail |
| IDX | B-tree | created_at, action | Report date range + action — audit report |

**Why audit-heavy indexing:** Append-only table grows monotonically; reporting and compliance queries dominate reads.

## Search indexes (Proposed)

| Target | Index approach | Why |
|--------|----------------|-----|
| borrowers | GIN trigram on full_name, phone | Global search — `searchService.mock.ts` |
| groups | GIN on display_name, system_id | Search entity GROUP |
| users | GIN on display_name, email | Search entity USER |
| audit_entries | GIN on action text **optional** | Search AUDIT_LOG entity |

**Status:** **Proposed** — no search backend today; architecture prepares for Phase 8.

## Reporting indexes (cross-table)

| Report | Driving indexes |
|--------|-----------------|
| Daily collection | payments(payment_date, collector_user_id) |
| Loan portfolio | loans(status), loan_schedule_weeks(due_date, status) |
| Defaulters | loan_schedule_weeks(status=MISSED/OVERDUE) |
| Collector performance | payments(collector_user_id, payment_date), reconciliation_submissions |
| Group risk | risk_flags(community, status), groups(status) |
| Financial ledger | financial_transactions(recorded_at), payments(recorded_at) |
| Audit log export | audit_entries(created_at, action) |

---

## Index anti-patterns (avoid)

| Anti-pattern | Why avoid |
|--------------|-----------|
| Index on audit_entries.reason | Low cardinality, rarely filtered alone |
| Over-indexing uploads.storage_key | Unique by construction if UUID paths |
| FK without index | All FK columns above get supporting indexes |
