## Decisions

1. **Append-only allocations** â€” Pool movements never update/delete rows; compensating entries in Phase 3.
2. **Denormalized aggregates** â€” `loan_pools` columns updated at seed/write time; allocations are source of truth for verification.
3. **UUID pool IDs** â€” Consistent with P14.3A; mock `PL-001` ids not used in API.
4. **`totalPoolFundsPesewas`** â€” API uses sum of pool `capitalPesewas` (not mock `DEMO_OPERATING_POOL_PESEWAS`).
5. **Read permission** â€” `VIEW_FINANCIAL_REPORTS` (same as loan list reads).
6. **Disburse integration deferred** â€” `loan_pool_id` set in seed only; no change to `disburseLoan` transaction.
