# P14.3A.3 — Environment Governance

**Phase:** P14.3A.3 Phase 1  
**Date:** 2026-06-09  
**Status:** Complete

---

## Strategy

| Principle | Implementation |
|-----------|----------------|
| Single source of truth | Code reads `process.env`; examples document all keys |
| Layered files | Root `.env` + app-specific `.env.local` overrides |
| No secrets in git | `.env.example` uses empty placeholders only |
| Production | Platform injects secrets; dotenv does not override existing vars |
| Frontend | Next.js native `.env*` loading (no extra package) |
| Backend | `apps/backend/src/config/load-env.ts` via `dotenv` |

---

## File Layout

```text
.env.example                          # Monorepo reference (all variables)
apps/frontend/.env.example            # Frontend subset
apps/frontend/.env.local.example      # Local dev overrides
apps/frontend/.env.production.example # Production deployment
apps/backend/.env.example             # Backend subset
apps/backend/.env.local.example       # Local secrets (DATABASE_URL)
apps/backend/.env.production.example  # Production deployment
```

**Developer workflow:**

1. `Copy-Item .env.example .env` at repo root
2. `Copy-Item apps/backend/.env.local.example apps/backend/.env.local` — set `DATABASE_URL`
3. `Copy-Item apps/frontend/.env.local.example apps/frontend/.env.local` — optional BFF URL
4. Run `npm run dev:api` and `npm run dev` — no manual shell exports

## Variable Reference (abbreviated)

### Database

| Variable | Required | Default | Local example | Production example |
|----------|----------|---------|---------------|-------------------|
| `DATABASE_URL` | Prod: yes | unset → in-memory | `postgresql://...@neon.tech/neondb?sslmode=require` | Neon pooled connection string |

### Backend API

| Variable | Required | Default | Local | Production |
|----------|----------|---------|-------|------------|
| `WILMS_API_PORT` / `API_PORT` | No | `4000` | `4000` | `4000` |
| `WILMS_API_HOST` | No | `127.0.0.1` | `127.0.0.1` | `0.0.0.0` |
| `WILMS_CORS_ORIGIN` | No | `http://127.0.0.1:3000` | Match frontend URL | `https://app.wilms.example` |

### Frontend API

| Variable | Required | Default | Local | Production |
|----------|----------|---------|-------|------------|
| `NEXT_PUBLIC_API_BASE_URL` | Prod API: yes | `""` → mock in dev | `http://127.0.0.1:3000/api/wilms` | `https://app.../api/wilms` |
| `WILMS_API_UPSTREAM` | No | `http://127.0.0.1:4000` | Same | `https://api.wilms.example` |
| `NEXT_PUBLIC_APP_URL` | No | — | `http://127.0.0.1:3000` | `https://app.wilms.example` |

### Uploads

| Variable | Required | Default | Notes |
|----------|----------|---------|-------|
| `UPLOAD_PROVIDER` | No | `local` | Set `cloudinary` in production |
| `CLOUDINARY_*` | When cloudinary | empty | All three credentials required |

### Mail / SMS

| Variable | Required | Default | Notes |
|----------|----------|---------|-------|
| `MAIL_PROVIDER` | No | `none` | `smtp` or `resend` when ready |
| `SMS_PROVIDER` | No | `none` | `arkesel` or `twilio` when ready |

### E2E

| Variable | Required | Default | Notes |
|----------|----------|---------|-------|
| `PLAYWRIGHT_BASE_URL` | No | derived from host+port | Full URL override |
| `PLAYWRIGHT_PORT` / `FRONTEND_PORT` / `E2E_PORT` | No | `3000` | Use `3001` to avoid dev conflict |

## Mock vs API Mode (unchanged business rule)

Development uses mock services unless `NODE_ENV=production` **and** `NEXT_PUBLIC_API_BASE_URL` is set. Documented in `data-provider/types.ts` — not modified in P14.3A.3.
