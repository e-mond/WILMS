# P14.3B — Demo Recovery Guide

**Phase:** 4D.2 Demo Certification  
**Date:** 2026-06-26  
**Purpose:** Recover from common demo failures without restarting the full certification cycle  
**Companion:** `P14.3B-demo-script.md`, `P14.3B-phase-4d2-recovery-check.md`

---

## Quick Recovery Matrix

| Symptom | Likely cause | Recovery | Time |
|---------|--------------|----------|------|
| Frontend shows mock data / wrong user | `NEXT_PUBLIC_USE_MOCK` unset or frontend not restarted | Set `false` in `.env.local`; **restart** `npm run dev` | 2 min |
| Proxy smoke fails login | Mock mode or frontend down | Restart frontend with live env; re-run smoke script | 3 min |
| Loan create 422 "already has active loan" | Neon seed exhaustion | Use existing loan narrative OR run `verify:live` | 5 min |
| Upload GET 500 | Stale server without uuidv7 fix | Restart backend; confirm `upload.service.ts` uses `uuidv7()` when DB enabled | 2 min |
| Reconciliation 422 duplicate | Date already submitted | Pick new cert date (`2032-MM-DD` pattern) | 1 min |
| Reconciliation 422 flagged variance | Physical ≠ expected without comment | Match physical to expected OR add ≥10 char comment | 1 min |
| Session expired on every action | Cookie not sent / mock login | Clear cookies; login via UI; confirm proxy URL | 2 min |
| 403 shown as session expired | Old frontend build | Confirm 4D.1 `apiClient.ts` deployed; rebuild frontend | 5 min |
| Backend 503 / health fail | API not running or Neon unreachable | Restart `npm run start -w @wilms/api`; check `DATABASE_URL` | 3 min |
| Reversal fails — no payment target | Seed exhaustion | `npm run cert:reversal:seed-reset -w @wilms/api` then retry | 10 min |

## Procedure R1 — Frontend live mode not active

**Symptoms:** Mock user IDs (e.g. `user-collector`); API calls never hit backend; proxy smoke fails.

**Root cause:** Default dev path uses mock provider unless `.env.local` overrides.

**Recovery:**

1. Confirm `apps/frontend/.env.local` contains:

```bash
NEXT_PUBLIC_USE_MOCK=false
NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:3000/api/wilms
WILMS_API_UPSTREAM=http://127.0.0.1:4000
```

2. **Stop and restart** frontend dev server (env vars load at startup only).
3. Hard-refresh browser (Ctrl+Shift+R) or clear site data for `127.0.0.1:3000`.
4. Verify:

```bash
node apps/frontend/scripts/proxy-reconciliation-smoke.mjs
```

Expected: **5/5 passed**.

**Evidence note:** Proxy smoke **failed on mock mode** before `.env.local` was applied in 4D.2 session; full path **pending frontend restart**.

## Procedure R2 — Loan create blocked (seed exhaustion)

**Symptoms:** HTTP 422 / VALIDATION — "This borrower already has an active or pending loan."

**Root cause:** Shared Neon seed — all demo borrowers have active or pending loans after certification runs.

**Recovery options (pick one):**

### Option A — Demo narrative (recommended for live demo)

1. Do not attempt inline create during demo.
2. Show existing active loan: list loans → open schedule → show payment history.
3. Cite carried evidence: `verify:live` 23/23 PASS, `workflow-create-loan` in same session.

### Option B — Re-run verify:live before demo

```bash
npm run verify:live -w @wilms/api
```

If 23/23 passes, loan lifecycle remains certified even if inline create fails again.

### Option C — Seed reset (destructive — avoid during stakeholder demo)

Only if isolated Neon branch available:

```bash
npm run cert:reversal:seed-reset -w @wilms/api   # reversal portfolio
# OR full db:seed on disposable branch
```

**Honest status:** Inline create **FALSE**; carried evidence **TRUE**.

## Procedure R3 — Upload failure

**Symptoms:** POST `/uploads` 201 but GET metadata/content 500; ID prefix mismatch.

**Root cause:** Pre-fix local upload IDs (`upload-…`) incompatible with PostgreSQL UUID column.

**Fix (4D.2):** `apps/backend/src/infrastructure/uploads/upload.service.ts` — `uuidv7()` when `isDatabaseEnabled()`.

**Recovery:**

1. Restart backend to pick up fix.
2. Re-test one upload:

```bash
npm run cert:upload:env -w @wilms/api
```

Expected: `active_provider: local`, PASS.

3. If still failing, check `.wilms-uploads/` directory writable.

**Cloudinary:** Not configured — do not attempt Cloudinary recovery; local provider is demo path.

## Procedure R4 — Reconciliation submit failure

**Symptoms:** HTTP 422 on submit; variance flagged.

**Recovery:**

| Error | Action |
|-------|--------|
| Duplicate date | Change date picker to unused date (e.g. random `2032-MM-DD`) |
| Flagged variance, missing comment | Enter physical cash = expected due OR add comment ≥10 chars |
| 403 permission | Wrong role — switch to Collector |
| 401 unauthorized | Re-login; check session cookie |

**Backend direct test (bypasses UI):**

```bash
npm run cert:demo:stakeholder -w @wilms/api
```

Check `reconciliation/*` steps in `P14.3B-phase-4d2-functional-evidence.json`.

## Procedure R5 — Reversal failure

**Symptoms:** Reversal prep error; cert-target syntax error in evidence.

**Recovery:**

1. Run dedicated reversal cert:

```bash
npm run cert:reversal:seed-reset -w @wilms/api   # if needed
npm run cert:reversal:functional -w @wilms/api
```

Expected: **10/10 PASS**.

2. In demo, use Super Admin account; select a recent payment with `REVERSIBLE` status.
3. If inline reversal in stakeholder cert fails, cite `cert:reversal:functional` carried evidence.

**Note:** Inline `audit-trail` step may show 0 entries — non-blocking when `successful-reversal` passes.

---

## Procedure R6 — Backend / database unavailable

**Symptoms:** Health check fails; login 500; Neon connection timeout.

**Recovery:**

1. Check `.env` / root `DATABASE_URL` valid.
2. Restart backend:

```bash
npm run start -w @wilms/api
```

3. Confirm health:

```bash
curl http://127.0.0.1:4000/health
```

4. If Neon branch suspended, wake branch in Neon console or switch to backup branch URL.

---

## Procedure R7 — Authentication / RBAC confusion

**Symptoms:** Officer can access admin routes; or all errors say "session expired."

**Recovery:**

| Issue | Fix |
|-------|-----|
| Wrong role logged in | Logout → login with correct demo user |
| 403 as session expired | Deploy 4D.1 frontend — distinct 401 vs 403 messages |
| Backend logout expected | No `/auth/logout` on API — use frontend logout only |

**Demo users:**

| Role | Email | Password |
|------|-------|----------|
| Super Admin | admin@wilms.demo | DemoAdmin1! |
| Loan Approver | approver@wilms.demo | DemoApprove1! |
| Collector | collector@wilms.demo | DemoCollect1! |
| Officer | officer@wilms.demo | DemoOfficer1! |

## Procedure R8 — Full stack reset (last resort)

**When:** Multiple cascading failures; demo window <10 minutes.

**Steps:**

1. Stop frontend and backend.
2. Restart backend → wait for health 200.
3. Restart frontend with `.env.local` live mode.
4. Run quick validation:

```bash
npm run verify:live -w @wilms/api          # 23/23
node apps/frontend/scripts/proxy-reconciliation-smoke.mjs  # 5/5
```

5. Resume demo from Step 2 (admin login) in demo script.

**Do not** run full `cert:demo:stakeholder` during live stakeholder session unless recovery window allows (~3 min).
