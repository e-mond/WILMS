## Payment Validation Detail

Payment uses schedule-aware inputs (not arbitrary amounts):

- Reads first unpaid row from `loan_schedules`
- `paymentDate` = schedule `due_date`
- `amountPesewas` = `decimalToPesewas(installment_amount)`
- `collectorId` = seeded collector user id

This aligns with `validatePaymentSubmission` in `apps/backend/src/modules/payments/service.ts`.
