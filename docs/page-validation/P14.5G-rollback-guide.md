# P14.5G Rollback Guide

**Release:** v0.2.0  
**Date:** 2026-06-29

---

## 1. Vercel (Frontend)

**Current production:** `wilms` project → `https://wilms.vercel.app`

```bash
# List recent deployments
vercel ls --prod

# Roll back to previous deployment ID
vercel rollback <deployment-url-or-id>
```

**Environment restore:** Revert `NEXT_PUBLIC_APP_URL` / `NEXT_PUBLIC_API_BASE_URL` in Vercel dashboard if domain change is rolled back.

---

## 2. Railway (API)

**Current service:** WILMS @ repo root (`Dockerfile`)

```bash
cd WILMS
railway logs --service WILMS
# Redeploy previous commit
git checkout <previous-sha>
railway up
```

Or use Railway dashboard → Deployments → Rollback.

---

## 3. Database (Neon)

**No destructive migrations in v0.2.0 release.**

- Schema changes: none new in this release branch beyond existing Drizzle journal
- **Do not** run `db:seed` on production to rollback
- Point-in-time restore: Neon console → Restore branch (requires Neon approval)

---

## 4. Git

```bash
git revert <release-commit-range>   # preferred
# OR deploy previous tag/branch without force-push to main
```

**Never** `git push --force` to `main`.

---

## 5. Recovery Verification

After rollback:

```bash
curl -s -o NUL -w "%{http_code}" https://wilms-production.up.railway.app/health
curl -s -o NUL -w "%{http_code}" https://wilms.vercel.app/login
WILMS_APP_URL=https://wilms.vercel.app WILMS_API_URL=https://wilms-production.up.railway.app npm run smoke:production
```

All must pass before declaring recovery complete.
