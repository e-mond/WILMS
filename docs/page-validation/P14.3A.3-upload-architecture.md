# P14.3A.3 — Upload Architecture

**Phase:** P14.3A.3 Phase 5  
**Date:** 2026-06-09  
**Status:** Complete

---

## Architecture Decision

```text
Files  → Cloudinary (production) | local disk (development)
Metadata → PostgreSQL schema (uploads table — existing)
API      → Express /uploads routes (unchanged contract)
```

---

## Implementation

```text
apps/backend/src/infrastructure/uploads/
├── types.ts              # UploadProvider interface, StoredUpload
├── config.ts             # UPLOAD_PROVIDER, CLOUDINARY_*, limits
├── validation.ts         # MIME + size validation
├── local-provider.ts     # LocalUploadProvider (dev default)
├── cloudinary-provider.ts# CloudinaryUploadProvider
├── index.ts              # getUploadProvider() factory
└── storage.ts            # Backward-compat re-export
```

### Provider selection

| `UPLOAD_PROVIDER` | Credentials | Active provider |
|-------------------|-------------|-----------------|
| `local` (default) | — | `LocalUploadProvider` |
| `cloudinary` | All `CLOUDINARY_*` set | `CloudinaryUploadProvider` |
| `cloudinary` | Missing creds | Falls back to local |

### Signed uploads

`GET /uploads/signature` (auth required) returns Cloudinary signed params when configured:

```json
{
  "data": {
    "cloudName": "...",
    "apiKey": "...",
    "timestamp": 1234567890,
    "signature": "...",
    "folder": "wilms"
  }
}
```

Uses `cloudinary.utils.api_sign_request`.

### Validation

- `UPLOAD_MAX_SIZE_BYTES` (default 10 MB)
- `UPLOAD_ALLOWED_MIME_TYPES` (comma-separated)

Applied in `POST /uploads` before save.

## API Contract (unchanged)

| Method | Path | Notes |
|--------|------|-------|
| POST | `/uploads` | dataUrl body — same schema |
| GET | `/uploads/:id` | Metadata |
| GET | `/uploads/:id/content` | Binary (local) or redirect fetch |
| POST | `/uploads/:id/delete` | Delete |
| GET | `/uploads/signature` | **New** — Cloudinary signed params |

No frontend/UI changes.
