## Upload contract

- Metadata in PostgreSQL; binary in external storage (`WILMS_UPLOAD_DIR` / future object storage).
- No base64 in DB (phase brief).
- `UPLOAD_PURPOSE` values stored as text matching frontend kebab-case slugs.

---

## RBAC contract

- `payments.collector_id` → `users.id` (ADR-001: collector is user extension).
- Permission checks continue via `@wilms/shared-rbac` — DB RBAC tables seeded to match.

---

## Verification outcome

**Approved to proceed** with P14.2 schema for: users, collectors, RBAC, borrowers, groups, payments, expenses, notifications, uploads, audit.

**Excluded (P14.3):** loans, pools, reconciliation, adjustments, risk flags.
