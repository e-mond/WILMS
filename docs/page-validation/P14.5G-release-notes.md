# P14.5G — Release Notes

**Version:** 0.2.1  
**Date:** 2026-06-20  
**Codename:** Production Hardening

---

## Highlights

- **CSRF protection** on all BFF mutation endpoints (login, logout, API proxy)
- **Expanded health endpoint** with version, uptime, runtime metadata, migration status
- **Version display** — `WILMS v0.2.1` on login, sidebar, dashboard footer, settings
- **Security headers** — explicit Helmet HSTS, hidePoweredBy, referrer policy
- **Demo user rotation script** for production credential hygiene

---

## Security

- Synchronizer token CSRF (`wilms_csrf` + `x-wilms-csrf`)
- Production smoke tests validate CSRF rejection and token issuance
- Demo rotation script: `apps/backend/scripts/rotate-production-users.mjs`

---

## API

- `/health` now includes `runtime.nodeVersion`, `runtime.deployedAt`, `runtime.buildId`
- `WILMS_DEPLOYED_AT` set at API startup

---

## Frontend

- Dashboard footer shows version label
- CSRF token auto-fetched before login
- Client bundles no longer import `node:crypto` (server/client split)

---

## Breaking Changes

None. Existing sessions remain valid; clients must obtain CSRF token before first mutation (handled automatically by `authService` and `apiClient`).

---

## Upgrade Notes

1. Merge PR and redeploy Railway + Vercel
2. Run demo-user rotation script (optional but recommended)
3. Update smoke test credentials if rotation performed
4. Verify `/health` returns v0.2.1
