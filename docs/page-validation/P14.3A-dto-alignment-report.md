# P14.3A — DTO Alignment Report

**Phase:** P14.3A

## Principle

Backend persists NUMERIC GHS + internal lifecycle; API responses match frontend TypeScript contracts exactly (camelCase pesewas integers, 5-value `LOAN_STATUS`).

---

## LoanDetail / LoanSummary

| Frontend field | API source | Alignment |
|----------------|------------|-----------|
| `id` | `loans.id` | ✅ UUID |
| `borrowerId` | `loans.borrower_id` | ✅ |
| `amountPesewas` | `decimalToPesewas(principal_amount)` | ✅ |
| `weeklyPaymentPesewas` | `decimalToPesewas(installment_amount)` | ✅ |
| `outstandingPesewas` | `decimalToPesewas(loan_balance)` | ✅ |
| `durationWeeks` | `duration_weeks` | ✅ |
| `status` | `external_status` enum | ✅ 5 values only |
| `paymentDay` | `payment_day` | ✅ |
| `startDate` | `start_date` | ✅ ISO date string |
| `cycleBatch` | `cycle_batch` | ✅ |

**Evidence:** `apps/backend/src/domain/loan/mappers.ts` — `mapLoanRowToDetail()`.

## LoanSchedule

```json
{ "loanId": "…", "weeks": [{ "weekNumber", "dueDate", "amountPesewas", "status" }] }
```

Week status values: `PENDING`, `PAID`, `MISSED` (OVERDUE not emitted).

**Evidence:** `mapScheduleRow()`; `getLoanSchedule()` in `loans/service.ts`.

## LoanProgressSummary

Ten fields replicated from `loan-progress.utils.ts`: `totalPaidPesewas`, `remainingBalancePesewas`, `percentRepaid`, `weeksCompleted`, `weeksRemaining`, `totalMissed`, `elapsedWeeks`, `paymentConsistencyScore`.

**Evidence:** `calculateLoanProgress()` in `mappers.ts`.

---

## LoanPaymentLogEntry

Disbursement + repayment entries from disbursement table and REPAYMENT ledger rows; amounts in pesewas; `type` = `DISBURSEMENT` | `REPAYMENT`.

**Evidence:** `listLoanPaymentLog()` in `loans/service.ts`.

---

## Payment POST Response

When DB enabled: `{ id, borrowerId, collectorId, loanId, amountPesewas, paymentDate, recordedAt, status: 'CONFIRMED', gps, weekNumber }`.

Legacy memory path retains `status: 'RECORDED'` when DATABASE_URL unset.

**Evidence:** `payments/service.ts`, `payments/routes.ts`.
