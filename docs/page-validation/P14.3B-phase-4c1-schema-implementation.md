## Schema Implemented

### `financial_reconciliations`

| Column | Type | Purpose |
|--------|------|---------|
| `id` | uuid PK | Submission identifier |
| `collector_user_id` | uuid FK → users | Submitting collector |
| `reconciliation_date` | text | ISO date `YYYY-MM-DD` |
| `expected_due_pesewas` | integer | Schedule-based expectation snapshot |
| `system_recorded_pesewas` | integer | CONFIRMED payments snapshot |
| `physical_cash_pesewas` | integer | Collector physical count |
| `primary_variance_pesewas` | integer | physical − expected |
| `collection_delta_pesewas` | integer | physical − system_recorded |
| `variance_class` | enum | BALANCED / SHORTAGE / OVERAGE |
| `variance_flagged` | boolean | Threshold exceeded |
| `threshold_percent` | integer | Policy snapshot at submit |
| `comment` | text nullable | Required when flagged (enforced in 4C.2) |
| `status` | enum | `SUBMITTED` (v1) |
| `submitted_at` | timestamptz | Immutable submit time |
| `created_at` | timestamptz | Row creation |

**Constraints:**
- Partial unique index `(collector_user_id, reconciliation_date) WHERE status = 'SUBMITTED'` — prevents duplicate submit (R-REC-05)
- FK `collector_user_id` → `users.id`

**Retention:** Indefinite; no soft-delete column (per architecture).

**File:** `apps/backend/src/db/schema/financial-reconciliations.ts`

### `reconciliation_history`

| Column | Type | Purpose |
|--------|------|---------|
| `id` | uuid PK | Event identifier |
| `reconciliation_id` | uuid FK | Parent submission |
| `event_type` | enum | SUBMITTED, COMMENT_ADDED |
| `actor_user_id` | uuid FK | Acting user |
| `before_snapshot` | jsonb nullable | Pre-change state |
| `after_snapshot` | jsonb | Post-change state (required) |
| `reason` | text nullable | Optional reason |
| `created_at` | timestamptz | Append-only timestamp |

**Constraints:** FK to `financial_reconciliations` and `users`.

**Append-only:** No update/delete columns — application must not mutate rows (4C.2).
