## Backend Must Provide

### 1. Core API host

- Base URL consumed by `apiClient` (env-configured).
- JSON DTOs matching `src/types/` (field names, enums, pesewas integers).
- Consistent error shape for UI toast handling.

### 2. Authentication & authorization

- Replace demo login with real IdP or credential store.
- Session/JWT compatible with middleware in `src/middleware.ts`.
- Role + permission claims matching `PERMISSION` constants used by `PermissionGate`.

### 3. Persistence

- Relational/document store for borrowers, loans, payments, groups, collectors, settings.
- Audit log append-only store.
- Notification inbox with read/unread state.

### 4. File storage

- Multipart upload endpoint (frontend `UploadFileInput` notes base64 is demo-only — `src/types/upload.ts`).
- Return `photoUrl` / `uploadId` on borrower, staff, and notification DTOs.

## Swap Procedure (when API ready)

1. Deploy API implementing contracts in `P13-api-contract-map.md`.
2. Set production env to API mode (`resolveDataProviderMode()` → `'api'`).
3. Point `apiClient` base URL at deployed host.
4. Run `npm run test` + `CI=1 npm run test:e2e` against API-backed staging.
5. Fix contract drift only in `src/services/*Service.ts` and types — **not** UI layouts per P13 constraint.
