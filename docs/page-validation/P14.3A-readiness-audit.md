## Status enums

### Frontend `LOAN_STATUS` (`types/loan.ts`)

`PENDING_DISBURSEMENT`, `ACTIVE`, `COMPLETED`, `DEFAULTED`, `WRITTEN_OFF`

### Spec lifecycle (internal)

`DRAFT` → `PENDING_APPROVAL` → `APPROVED` → `DISBURSED` → `ACTIVE` → `COMPLETED` / `DEFAULTED` / `WRITTEN_OFF`

### Mapping strategy (approved)

| Internal lifecycle | API `status` (frontend) |
|--------------------|-------------------------|
| DRAFT, PENDING_APPROVAL, APPROVED | `PENDING_DISBURSEMENT` |
| REJECTED | Excluded from lists; lifecycle only |
| DISBURSED, ACTIVE | `ACTIVE` |
| COMPLETED | `COMPLETED` |
| DEFAULTED | `DEFAULTED` |
| WRITTEN_OFF | `WRITTEN_OFF` |

`POST /loans` matches mock: creates `PENDING_DISBURSEMENT` + schedule (skips draft for frontend parity). `PATCH approve|reject` operate on internal lifecycle for admin workflow.

- `loans.version`, `loan_schedules.version`, `payments.version` (existing)
