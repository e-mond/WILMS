**Date:** 2026-06-15  
**Scope:** Verified axe/WCAG violations from `P13.1-e2e-run-v3.log` only. No visual redesign.

## Method

- Parsed axe JSON in v3 E2E log for violation `id`, `target`, and `html` snippets.
- Fixed defects in application code; adjusted E2E ready selectors where failures were test drift (not product a11y).
- Re-validated via `CI=1 npm run test:e2e` (`accessibility.spec.ts`) ‚Äî see v4 log.

## Verified Violations Fixed

### 1. `button-name` (critical) ‚Äî Settings control

**Evidence (v3 log):** `target: a[aria-label="Settings"] > .bg-transparent.transition-opacity.h-8`  
**File:** `src/components/layout/shell/navbar/ShellNavbarActions.tsx`  
**Issue:** Nested interactive `Button` inside `Link` ‚Äî inner button had no accessible name.  
**Fix:** Replaced with a single `Link` styled as icon control with `aria-label="Settings"`.

### 2. `color-contrast` (serious) ‚Äî Online live badge

**Evidence (v3 log):** `#9cb586` on `#f4f9ee`, contrast 2.09:1; element: `text-status-active` on `bg-status-active-light` in `AppNavbar`.  
**File:** `src/components/layout/shell/AppNavbar.tsx`  
**Fix:** Badge label uses `text-text-primary` on existing `bg-status-active-light` surface (preserves badge shape/copy "Online").

### 2b. `color-contrast` (serious) ‚Äî status semantic tokens (v4 log)

**Evidence (v4 log):** `text-status-at-risk` (#ba7517) and `text-status-info` (#2563eb) below 4.5:1 on card/background surfaces across dashboard and management pages.  
**File:** `src/styles/tokens.css` (light mode only)  
**Fix:** Darkened `--color-status-at-risk` to `#8B5A14`, `--color-status-info` to `#1D4ED8`, aligned `--color-warning` with at-risk.

### 3. `aria-allowed-attr` (critical) ‚Äî selectable data table rows

**Evidence (v3 log):** `<tr role="button" aria-selected="true">` on groups/collectors/loan-pools pages.  
**File:** `src/components/data-display/DataTable.tsx`  
**Issue:** `aria-selected` is not valid on `role="button"`.  
**Fix:** Use `aria-pressed={isSelected}` for selectable rows.

### 4. Duplicate dialog landmarks (E2E blocker, a11y tree noise)

**Evidence (v3 log):** strict-mode duplicate `role="dialog"` named `Search WILMS` and `Notifications`.  
**Files:** `ShellNavbarActions.tsx`, `ShellMobileOverflowMenu.tsx`, new `ShellOverlayPanels.tsx`, `DashboardShell.tsx`  
**Fix:** Mount search and notification panels once per shell.
