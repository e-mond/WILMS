## Health Endpoint

**URL:** `GET /health` (Railway API, no auth)

| Field | Purpose |
|-------|---------|
| `status` | `ok` / degraded |
| `version` | From root `package.json` |
| `migrations` | `expected`, `applied`, `status` |
| `database.connected` | Neon connectivity |
| `uploads.cloudinaryConfigured` | Upload provider |
| `runtime.nodeVersion` | Node version |
| `runtime.deployedAt` | Deploy timestamp |
| `runtime.buildId` | Railway build UUID |
| `runtime.uptimeSeconds` | Process uptime |

Implementation: `apps/backend/src/modules/health/health.service.ts`  
Smoke: `production-smoke.ts` validates 200 + envelope.

## Platform Monitoring

### Railway (API)

| Capability | Notes |
|------------|-------|
| Deploy logs | `railway logs` or dashboard |
| Metrics | CPU/memory in Railway dashboard |
| Alerts | Configure in Railway project settings |
| Health probe | External curl or smoke pipeline post-deploy |

### Vercel (Frontend)

| Capability | Notes |
|------------|-------|
| Deployment status | Vercel dashboard / `vercel ls` |
| Analytics | Vercel Web Analytics (if enabled) |
| Function logs | Edge/serverless invocation logs |
| Preview deploys | Per-PR previews when Git integration active |

## CI/CD Verification Pipeline

| Stage | Check |
|-------|-------|
| Staging (`deploy-staging.yml`) | Health curl + `smoke:staging` |
| Production (`deploy-production.yml`) | Baseline health → deploy → version consistency → `smoke:production` |

Gated by repo variables: `ENABLE_STAGING_DEPLOY`, GitHub Environment `production` with required reviewers.

## Gaps

| Gap | Recommendation |
|-----|----------------|
| No APM (Datadog/New Relic) | Add at v1.0 if SLA required |
| No uptime external monitor | Configure UptimeRobot/Pingdom on `/health` |
| Structured log aggregation | Railway log drain to external sink |

## Verdict

**ADEQUATE FOR v0.2.2** — health endpoint comprehensive; platform-native monitoring via Railway/Vercel dashboards. External uptime monitoring recommended before stakeholder rollout.
