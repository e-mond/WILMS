# P14.1D — Workspace Decision

**Phase:** P14.1D  
**Date:** 2026-06-09  
**Status:** Implemented

---

## Decision

**Use npm workspaces** (not pnpm).

## Rationale

| Option | Assessment |
|--------|------------|
| **pnpm workspaces** (preferred in phase brief) | `pnpm` is not installed on the development machine. Adding a global toolchain dependency would block immediate validation. |
| **npm workspaces** (chosen) | Already in use via root `package-lock.json`; zero additional tooling; supports `npm run dev -w @wilms/frontend` pattern. |

## Configuration

Root `package.json`:

```json
{
  "workspaces": ["apps/*", "packages/*"]
}
```

## Workspace members

| Package | Path | Purpose |
|---------|------|---------|
| `@wilms/frontend` | `apps/frontend/` | Next.js UI |
| `@wilms/api` | `apps/backend/` | Express API |
| `@wilms/shared-contracts` | `packages/shared-contracts/` | Canonical enums |
| `@wilms/shared-rbac` | `packages/shared-rbac/` | Permissions and roles |
| `@wilms/shared-types` | `packages/shared-types/` | Shared DTO / API types |
| `@wilms/shared-validation` | `packages/shared-validation/` | Shared Zod schemas |
| `@wilms/shared-utils` | `packages/shared-utils/` | Pure utilities |

## Turborepo

`turbo.json` is present for future pipeline orchestration. Root scripts currently invoke npm workspace commands directly; turbo is optional and not required for P14.1D validation.

## Future migration to pnpm

If pnpm is adopted later:

1. Add `pnpm-workspace.yaml` mirroring npm workspace globs.
2. Remove root `package-lock.json`; generate `pnpm-lock.yaml`.
3. Replace `-w` flags with `--filter` in root scripts.

No application code changes required.
