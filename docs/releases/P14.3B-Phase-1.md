# P14.3B Phase 1 Release Notes — Loan Pools

**Milestone:** P14.3B  
**Branch:** `feature/p14.3b-phase-1`  
**Date:** 2026-06-09

---

## Features

- PostgreSQL schema: `loan_pools`, `pool_memberships`, `pool_allocations`
- Migration `0003_loan_pools` with indexes on status and pool_id FKs
- Nullable `loans.loan_pool_id` for disburse attribution
- Read API: `GET /loan-pools`, `GET /loan-pools/:id`
- Seed: 3 pools with reconciled allocation history
- Verification: `npm run verify:pools -w @wilms/api` (5 checks)

---

## Architecture Decisions

1. **Append-only allocations** — pool capital movements are immutable rows; corrections use compensating `ADJUSTMENT` type (writers in Phase 2+).
2. **Denormalized aggregates** on `loan_pools` for dashboard reads; verified against allocation sums.
3. **Read-only Phase 1** — no HTTP mutations; avoids premature disburse integration.
4. **RBAC:** `VIEW_FINANCIAL_REPORTS` matches loan list read pattern.

## Known Limitations

- No `appendAuditEntry` for pool operations (enum only)
- No soft delete on `loan_pools`
- `pool_memberships` empty (groups not in DB seed)
- Disburse/payment do not write live allocations
- API `totalPoolFundsPesewas` ≠ mock fixed KPI constant
