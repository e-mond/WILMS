### Implemented improvements

#### E-001 — Sanitize 500 error responses

- **Files:** `apps/backend/src/http/error-handler.ts`
- **Change:** Return `"An unexpected error occurred."` instead of internal `error.message` on 500
- **Why low-risk:** Single file; no API contract change for success paths; clients still receive `{ error: { message, code } }` shape
- **Verify:** Trigger unhandled error; response must not contain stack trace or file paths

#### E-002 — Live mode uses backend-signed session cookie

- **Files:** `apps/frontend/src/lib/auth/authenticate.ts`, `apps/frontend/src/app/api/auth/login/route.ts`, `apps/frontend/src/lib/auth/session.ts`
- **Change:** Cookie stores backend `token`; session decode handles `payload.signature` format for UI expiry
- **Why low-risk:** No schema/API contract change; fixes auth mismatch after F-S001
- **Verify:** Login with `NEXT_PUBLIC_USE_MOCK=false`; API calls succeed via BFF Bearer

### Summary

One **stability** and one **auth-integration** low-risk improvement were implemented. Performance, E2E, and documentation alignment items are **deferred** to 5B or post-production per low-risk criteria.
