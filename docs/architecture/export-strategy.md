# WILMS Export & Reporting Standard
> Official document export, print, and CSV strategy for all WILMS modules

**Status:** Adopted 2026-06-09  
**Implementation:** `src/features/export/`

---

## 1. Purpose

All WILMS exports must be suitable for:

- Executive review
- Board meetings
- Regulatory review
- Internal audits
- Field operations
- Official record keeping

**No page may implement custom export formatting independently.** Use the shared export framework.

---

## 2. Framework Architecture

```text
src/features/export/
├── types.ts                    # WilmsExportDocument, metadata, sections
├── constants/branding.ts       # Colours, org identity, workbook title
├── builders/                   # Domain → WilmsExportDocument mappers
├── engines/
│   ├── csv-engine.ts           # Branded CSV with metadata preamble
│   ├── excel-engine.ts         # Multi-sheet ExcelJS workbooks
│   ├── pdf-engine.ts           # jsPDF + autotable official documents
│   └── print-engine.ts         # Dedicated print HTML templates
├── components/
│   ├── ExportCsvButton.tsx     # Standard list/report CSV export
│   └── WilmsExportActions.tsx  # CSV / Excel / Print / PDF action row
└── hooks/useWilmsExportActor.ts
```

## 3. Report Identification

Every export must include:

| Field | Example |
|---|---|
| Report ID | `WILMS-GRP-2026-000123` |
| Generated timestamp | `14 May 2026, 09:41 GMT` |
| Generated by | Session user display name |
| Environment | Development / Staging / Production |
| WILMS branding | Logo block, full org name, identifier |

Report IDs use prefix by type: `GRP`, `BOR`, `COL`, `POL`, `AUD`, `RSK`, `FIN`, etc.

## 4. Format Requirements

### PDF

- Official header: WILMS logo text, full name, org identifier, report title, report ID, timestamp, actor, environment
- Executive summary section
- Branded tables (alternating rows, header styling)
- Footer: page numbers, report ID, confidentiality notice
- A4 portrait/landscape based on column count
- Brand colours: Primary `#0F6E56`, Accent `#BA7517`, Danger `#993C1D`, Success `#3B6D11`

### Print

- **Never** call `window.print()` on the current app page
- Use `printWilmsDocument()` which renders a dedicated HTML template in a **hidden iframe** (no popup windows)
- `printWilmsDocument()` returns `WilmsPrintResult` — it **never throws**; callers show a toast when `{ ok: false }`
- Failure reasons: `iframe_unavailable`, `print_blocked`, `unknown` — always degrade to PDF download guidance
- `openWilmsPrintPreview()` uses an in-page overlay (not `window.open`) for optional preview
- Remove navigation, sidebar, buttons, filters from print templates
- Layout mirrors PDF structure (DM Sans / DM Serif Display)

### Excel

- Workbook title: `WILMS Report`
- Sheets: Executive Summary, Detailed Records, Statistics (if metrics), Audit Information
- Header row styling, frozen headers, filters, auto-width columns
- Currency / percentage / date formatting in cell values

### CSV

- Machine-readable with metadata comment preamble (`# Report ID: ...`)
- Human-readable column headers (`Borrower ID`, not `borrower_id`)
- Proper currency labels (`Loan Amount (GHS)`)
- Executive summary and audit sections included

## 5. Usage

### List / report CSV

```tsx
<ExportCsvButton
  reportType={WILMS_REPORT_TYPE.GROUP_LIST}
  reportTitle="Groups Management Export"
  filename="groups.csv"
  headers={['Group ID', 'Group Name']}
  rows={csvRows}
/>
```

### Full profile export (CSV, Excel, Print, PDF)

```tsx
const document = buildGroupProfileExportDocument(group, generatedBy);
<WilmsExportActions document={document} filenameBase={`${group.id}-profile`} />
```

### Programmatic export

```ts
import { exportWilmsDocument } from '@/features/export';
await exportWilmsDocument(document, 'pdf', 'report.pdf');
```

## 6. Migration Rule

Legacy utilities in `src/utils/export-csv.ts` and `src/utils/export-group-profile.ts` re-export from the framework. New code must import from `@/features/export` only.

| Collectors | `COLLECTOR_LIST` | `ExportCsvButton` |
| Loan pools | `LOAN_POOL` | `ExportCsvButton` |

| Reports index | `GENERIC_REPORT` | `WilmsExportActions` |

| Financial reports | `FINANCIAL_LEDGER`, etc. | `ExportCsvButton` |

All future export functionality must extend this table and use the framework.
