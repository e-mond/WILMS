# V1.2.2 Security Report

**Release:** 1.2.2  
**Branch:** `hotfix/v1.2.2-security-user-lifecycle`

## Scope

Security and user lifecycle stabilization across authentication, session management, user deletion, and collector administration fee workflow.

## Changes

### Session invalidation

- Added `users.session_version` (migration `0018_v122_security_user_lifecycle.sql`).
- Login tokens include `sessionVersion`.
- `requireAuth` validates session version, role, and account status on every protected request.
- Sessions are invalidated when users are suspended, deleted, or their role changes.

### Permanent user deletion

- Invited users without activity are hard-deleted.
- Active users with history are anonymized: credentials removed, email replaced, display name scrubbed, `deletedAt` set.
- Auth artifacts purged: OTP challenges, password reset tokens, push subscriptions, notification preferences, in-app notifications, role assignments.

### Collector administration fee

- Fees persist in `borrower_admin_fees` (no in-memory loss on API restart).
- Login never triggers an administration fee dialog.
- Fees are recorded per approved borrower before loan disbursement only.

## Risk assessment

| Area | Before | After |
|------|--------|-------|
| Stale sessions after suspend | Token valid until expiry | Revoked on next request |
| Soft-deleted users recoverable | Yes | Invited hard-deleted; active anonymized |
| Admin fee data loss on restart | Yes (in-memory) | Persisted in database |

## Deployment

1. Run migration `0018_v122_security_user_lifecycle.sql` on production Postgres.
2. Deploy API and frontend at version `1.2.2`.
3. Re-run `npm run smoke:production` and `npm run smoke:rbac`.
