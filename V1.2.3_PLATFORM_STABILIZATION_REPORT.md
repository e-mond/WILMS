# WILMS v1.2.3 Platform Stabilization Report

**Version:** 1.2.3  
**Branch:** `cursor/v123-platform-stabilization-8847`

## Summary

Release 1.2.3 completes the invitation lifecycle, hardens SMS invitation delivery, improves production UI stability, and expands permanent deletion coverage. The platform retains all v1.2.2 security controls including session invalidation and persisted admin fees.

## Key Changes

### Invitation lifecycle
- Database columns: `invited_at`, `accepted_at`, `first_login_at`
- Accept-invitation link records acceptance before login
- First successful login records acceptance and first-login timestamps
- Onboarding (`complete-onboarding`) sets `ACTIVE` with audit `user.activated`
- Settings User Management shows **Invited**, **Accepted**, **Pending setup**, or **Active**

### SMS invitations
- Centralized Ghana phone normalization and validation
- Invitation SMS results tracked alongside email (`invitationSmsStatus`)
- Delivery failures logged to `message_deliveries` with meaningful errors
- Test SMS uses the same normalization path

### Production stability
- Blob URL lifecycle fixed in uploads and registration review
- DOCX export defers `revokeObjectURL` until after download starts
- `StatusBadge` tolerates unknown statuses
- User profile modal handles errors and null arrays safely
