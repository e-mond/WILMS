# WILMS v1.3.6 Release Candidate Report

**Version:** 1.3.6-rc1

**Type:** Production Stabilisation (no new features)

## Summary

v1.3.6-rc1 addresses verified production issues: duplicate collector PIN settings, mock-data guard, health diagnostics, admin collector messaging validation, and collector display id formatting. Feature development is frozen; only bug fixes and operational hardening are included.

---

## Issues resolved

| Issue | Root cause | Fix | Evidence |
|-------|------------|-----|----------|
| Duplicate PIN / App Lock on collector Settings | Both `PIN` and `APP_LOCK` sections rendered `AppLockSetupPanel` | Removed `PIN` from collector settings nav | `role-settings-sections.ts` |
| Legacy security page duplication | `/collector/security` duplicated App Lock UI | Redirect to `/collector/settings` | `collector/security/page.tsx` |
| Mock data in production risk | Webpack could resolve mock services without API URL at build | `shouldUseMockServices()` returns false when `NODE_ENV=production` | `next.config.mjs` |
| Health `degraded` | Live probe: migrations 22/23; missing tables from `0020` | Added `degradedReasons`; runbook updated — **operational fix: run migrations** | `curl /health` 2026-07-12 |
| Open messages error on Collectors page | `collectorId` Zod schema required UUID; demo ids are `user-collector` | Relaxed to `z.string().min(1).max(128)` | `messages/routes.schema.test.ts` 3/3 PASS |
| Collector aside display id | Showed raw `user.id` UUID | Use `resolveCollectorDisplayId()` | `CollectorsAsidePanel.tsx` |
