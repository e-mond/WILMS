# WILMS v1.1.1 Production Hotfix Report

**Branch:** `hotfix/v1.1.1-production-fixes`  
**Date:** 2026-07-05  
**Package version:** `1.1.1`

---

## Executive summary

Production verification on v1.1 (`d2a64bb`) confirmed several UX and data-display defects. This hotfix addresses verified issues only — no new features. Railway production health was **200 OK** at verification time; deploy-sync matched production commit `d2a64bb` before hotfix deployment.

---

## Root causes and fixes

### 1. Registration — Home/Business address inputs (React #185)

| Item | Detail |
|------|--------|
| **Root cause** | `registrationSessionId` used `Date.now()` on every render, producing unstable `entityId` values. Autosave used `watch()` return value as a `useEffect` dependency, retriggering on every keystroke and compounding re-renders. |
| **Fix** | Stable session id via `useRef` + `crypto.randomUUID()`. Autosave converted to `watch()` subscription with debounce. Identity/guarantor effects use scalar field deps instead of array references. Label updated to **Home address**. |
| **Files** | `apps/frontend/src/features/borrower-registration/components/BorrowerRegistrationWizard.tsx` |

### 2. Navigation highlighting (Reports + Collections)

| Item | Detail |
|------|--------|
| **Root cause** | `/reports/daily-collection` matched both Collections nav and parent `/reports` prefix rule. |
| **Fix** | `exact: true` on Reports, Audit Log, Settings. `isShellNavLinkActive` excludes `/reports/*` child routes from parent `/reports` match. |
| **Files** | `apps/frontend/src/constants/navigation.ts`, `apps/frontend/src/layouts/ShellNavLink.tsx`, `apps/frontend/src/tests/layouts/shell-nav-link.utils.test.ts` |

### 3. Audit log — empty action labels

| Item | Detail |
|------|--------|
| **Root cause** | DB repository lowercased actions (`borrower.registered`) while UI labels keyed on `BORROWER_REGISTERED`. Login events mapped to `SETTINGS_EXPORTED`. Several loan/reversal actions missing from frontend label map. |
| **Fix** | Repository returns enum values unchanged. `normalizeAuditAction()` on append. Correct `USER_LOGGED_IN` / `USER_LOGIN_FAILED` / `SETTINGS_UPDATED` mappings. Expanded `AUDIT_ACTION` labels. Settings updates write audit entries. |
| **Files** | `apps/backend/src/repositories/audit.repository.ts`, `apps/backend/src/infrastructure/audit/audit-log.ts`, `apps/backend/src/db/schema/enums.ts`, `apps/backend/src/db/migrations/0013_v111_hotfixes.sql`, `apps/backend/src/modules/settings/service.ts`, `apps/frontend/src/constants/audit.ts`, `apps/frontend/src/constants/audit-display.ts` |

### 4. Disbursements — readable IDs (`DIS-YYYY-NNNNNN`)

| Item | Detail |
|------|--------|
| **Root cause** | `loan_disbursements` stored only UUID `id`; UI/API showed raw UUIDs. |
| **Fix** | Added `display_id` column (migration + backfill). `formatDisbursementDisplayId()` in shared-utils. Generated on insert. Exposed in payment log API and UI. |
| **Files** | `packages/shared-utils/src/display-ids.ts`, `apps/backend/src/db/schema/loan-disbursements.ts`, `apps/backend/src/repositories/loan-disbursement.repository.ts`, `apps/backend/src/modules/loans/service.ts`, `apps/frontend/src/types/loan.ts`, `apps/frontend/src/utils/entity-display-id.ts`, `apps/frontend/src/features/loan-management/components/LoanPaymentLogTable.tsx` |

### 5. Disbursements — loans stuck PENDING

| Item | Detail |
|------|--------|
| **Root cause** | Backend `/loans/:id/disburse` existed but **no production UI** called it. `LoanDetailPanel` showed UUID title with no disburse action. |
| **Fix** | Added `useDisburseLoan` hook and **Disburse loan** button on loan detail for `PENDING_DISBURSEMENT` loans (permission-gated). Documented lifecycle: create → pending disbursement → manual disburse → active. |
| **Files** | `apps/frontend/src/features/loan-management/hooks/useDisburseLoan.ts`, `apps/frontend/src/features/loan-management/components/LoanDetailPanel.tsx` |

**Intended disbursement lifecycle:** Loan created → `PENDING_DISBURSEMENT` → admin fee confirmed → super-admin disburses → `ACTIVE` with ledger + `DIS-*` record.

### 6. Disbursements — filter controls

| Item | Detail |
|------|--------|
| **Root cause** | Status filter used pill buttons (poor mobile/keyboard UX). |
| **Fix** | Replaced with `<Select>` dropdowns for cycle and status. CSV export uses readable loan IDs. |
| **Files** | `apps/frontend/src/features/loan-management/components/LoanPortfolioList.tsx`, `apps/frontend/src/tests/loan-management/LoanPortfolioList.test.tsx` |

### 7. Group members — readable borrower IDs

| Item | Detail |
|------|--------|
| **Root cause** | Group member API/UI exposed raw `borrowerId` UUIDs. |
| **Fix** | Backend `buildMembers()` adds `displayId` via `formatBorrowerDisplayId`. UI shows readable ID; search includes `displayId`. |
| **Files** | `apps/backend/src/modules/groups/service.ts`, `apps/frontend/src/types/group.ts`, `apps/frontend/src/features/group-management/components/profile/GroupMembersSection.tsx`, `apps/frontend/src/services/mock/group-members.resolver.ts` |

### 8. Loan pool create form

| Item | Detail |
|------|--------|
| **Root cause** | Raw labels/inputs without `FormField`, no form submit semantics, capital field not `type="number"`. |
| **Fix** | Wrapped in `<form>`, `FormField` labels, `autoFocus`, submit via form id, numeric capital input. |
| **Files** | `apps/frontend/src/features/loan-pools/components/LoanPoolsPanel.tsx` |

### 9. Toggle buttons (Switch)

| Item | Detail |
|------|--------|
| **Root cause** | Square toggles, inconsistent sizing, label not linked for keyboard users. |
| **Fix** | Pill track/thumb, 44px touch target, `aria-labelledby`, `loading` + `aria-busy`, disabled styling. |
| **Files** | `apps/frontend/src/components/ui/Switch.tsx` |

### 10. Dialog accessibility (Radix warning)

| Item | Detail |
|------|--------|
| **Finding** | No `@radix-ui/react-dialog` in codebase. Custom `Modal`/`Drawer` already expose `h2` + `aria-labelledby`. Executive mobile drawer uses sr-only title when `hideHeader`. No code change required beyond existing components. |
